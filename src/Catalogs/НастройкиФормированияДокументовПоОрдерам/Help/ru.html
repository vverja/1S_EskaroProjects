Справочник «Настройки формирования документов по ордерам»

Справочник предназначен для хранения настроек формирования документов «Поступление товаров и услуг» и «Реализация товаров и услуг» по данным документов «Приходный ордер на товары» и «Расходный ордер на товары».

Формирование документов по настройке может выполняться

  • в обработке «Формирование документов по ордерам»
  • автоматически, регламентным заданием

Настроек формирования документов может быть несколько:

  • для разных видов ордеров (приходный / расходный) формируются разные виды документов («Поступление товаров и услуг» либо «Реализация товаров и услуг») 
  • для разных организаций могут формироваться документы, отражаемые в разных видах учета
  • для разных контрагентов могут использоваться разные шаблоны документов

Параметры отбора ордеров

В поле «Начиная с» указана дата, начиная с которой следует обрабатывать данные. Её значение

  • изменяется автоматически по мере формирования документов
  • может быть изменено вручную (по кнопке выбора даты)
    При начале работы с механизмом необходимо вручную установить эту дату в соответствии с датой начала использования механизма.

В табличном поле необходимо указать параметры отбора обрабатываемых ордеров. 
Дополнительно к предопределенным параметрам отбора можно добавить новые по реквизитам шапки ордера (Контрагент, Заказ поставщику и т.д.)

Обязательным является параметр «Направление»: определяет вид обрабатываемого ордера (приходный / расходный).

Независимо от выполненных настроек при формировании документов не учитываются следующие ордера:

  • непроведенные
  • приходные ордера
    • с установленным признаком «Без права продажи»
    • с видами операции «от подотчетника» либо «перемещение»
  • расходные ордера с видом операции, отличным от «не определен»

Параметры создаваемых документов

Вид документа: выбирается из списка, обязателен к заполнению

Шаблон документа: указывается шаблон, в соответствии с которым будут заполняться реквизиты шапки новых документов. Не обязателен к заполнению.
Если шаблон не указан, реквизиты шапки будут заполняться по реквизитам ордера (например, Контрагент - непосредственно из ордера, Договор - основной договор контрагента, Тип цен - из договора и т.д.).

Периодичность

Документы будут сформированы с заданной периодичностью. Каждый документ будет включать данные за период

  • не больший, чем период детализации
  • относящийся к одному календарному месяцу

Документы формируются датой последнего дня периода, данные за который содержат.

Например:
Детализация - 1 документ в неделю. Формируются документы по данным за период с 22 января (понедельник) по 04 февраля.
Будет сформировано 3 документа:
  1. включающий данные за период с 22 по 28 января, дата документа - 28 января
  2. включающий данные за период с 29 по 31 января, дата документа - 31 января
  3. включающий данные за период с 01 по 04 февраля, дата документа - 04 февраля

Расписание

Для автоматического формирования документов следует установить флаг «Использовать расписание» и ввести расписание.

В момент записи редактируемой Настройки будет создано связанное с ней регламентное задание.

«Не обрабатывать документы за предыдущие...»: позволяет указать, за какое количество дней относительно текущей даты не следует обрабатывать документы.

При каждом запуске по расписанию формирование документов будет выполняться за период

с даты, указанной в поле «Начиная с»

по 

    • момент запуска
    • одну из дат, предшествующих дате запуска
