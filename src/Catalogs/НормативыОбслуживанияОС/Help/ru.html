Справочник «Нормативы обслуживания ОС»

Справочник содержит нормативы для выполнения работ по обслуживанию. Информация о нормативе обслуживания может иметь несколько состояний утверждения. Текущее состояние указывается в реквизите «Состояние». По факту утверждения норматива состояние должно иметь значение «Утвержден» и дату утверждения.

Суть работы по обслуживанию выбирается из справочника «Номенклатура» в реквизите «Обслуживание». Количество итераций по обслуживанию указывается в реквизите «Количество».

Нормативы по обслуживанию могут быть двух типов:

  • периодические - такой норматив означает, что работу по обслуживанию, указанную в нормативе следует проводить периодически, с равными интервалами. В этом случае в нормативе следует указать периодичность обслуживания и число периодов между обслуживанием;
  • по выработке - обслуживание производится только при достижении некоторыми параметрами критических значений. Такими параметрами могут быть, например, число сбоев в работе, пробег, количество выполненных технологических операций. Для задания такого типа норм следует указать параметр выработки и значение параметра, при достижении которого должно проводиться обслуживание.

Данные нормативов используются при анализе оборудования, требующего обслуживания. Такой анализ выполняется с помощью обработки «Формирование заказов на обслуживание ОС».
