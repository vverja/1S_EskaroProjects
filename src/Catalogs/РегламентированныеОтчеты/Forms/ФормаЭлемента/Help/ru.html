Справочник "Регламентированные отчеты", форма элемента

Каждый элемент справочника "Регламентированные отчеты" содержит ссылку либо на один из объектов конфигурации, либо на внешнюю обработку системы "1С:Предприятие 8.0", реализующих функции того или иного регламентированного отчета.

При воде нового элемента в справочник необходимо заполнить реквизит "Наименование" - это краткое наименование формы отчетности, которое будет выводится в списке регламентированных отчетов.

Переключатель "Использовать" служит для того, чтобы указать, какая форма регламентированного отчета будет использоваться: либо объект конфигурации, либо внешняя обработка "1С:Предприятия 8.0".

При выборе переключателя "объект" необходимо в поле ввода указать имя объекта конфигурации, который будет использоваться в качестве регламентированного
отчета. Имя объекта следует указывать так, как оно указано в Конфигураторе.

При выборе переключателя "файл" в поле ввода следует указать файл внешней обработки "1С:Предприятие 8.0". Подробнее об использовании внешних отчетов см.
ниже.

В поле "Описание" можно указать подробное описание формы отчетности, например, полное неаименование формы отчетности. Описание, если оно заведено, выводится для выбраного отчета в нижней части списка регламентированных отчетов.

После нажатия кнопки "Записать" изменения в элементе справочника будут сохранены, при этом:

  • поле для указания объекта конфигурации будет сделано недоступным;
  • внешний отчет будет загружен в информационную базу, а вместо полного имени отчета появится текст "Отчет загружен в ИБ".

После нажатия кнопки "OK" выполняются те же действия, но форма элемента справочника при этом будет закрыта.

Использование внешних отчетов

В типовых конфигурациях для 1С:Предприятия 8.0, выпускаемых фирмой "1С", формы регламентированной отчетности входят в состав конфигураций и выпускаются вместе с релизами конфигураций.

Однако, когда возникает необходимость, в качестве регламентированного отчета вместо объекта конфигурации можно использовать файл внешней обработки 1С:Предприятия 8.0. Такие файлы могут распространяться фирмой "1С" или ее партнерами в регионах.

Использование внешних отчетов является временным решением, которое следует использовать только до выхода релиза конфигурации, в котором будет
встроены необходимые изменения. Поэтому на использование внешних отчетов накладываются определенные ограничения:

  • при выпуске внешнего отчета оговаривается, с каким релизом конфигурации он может использоваться;
  • не гарантируется поддержка  внешним отчетом предыдущих релизов конфигурации;
  • при работе пользователя с конфигурацией нельзя использовать более одной версии внешнего отчета;
  • открытие формы отчета выполняется более продолжительное время, по сравнению с отчетом реализованным как объект конфигурации;
  • выгрузка данных отчета в электронном виде выполняется намного медленнее
  • и другое.

Для использования внешнего отчета пользователю необходимо указать, что для данного регламентированного отчета будет использоваться не внутренний объект
конфигурации, а внешний отчет. Для этого в справочнике "Регламентированные отчеты" необходимо открыть на редактирование элемент справочника, и в группе
"Использовать" выбрать переключатель "файл". После этого необходимо указать, какой файл следует использовать в качестве внешнего отчета, для этого
необходимо нажать кнопку "Выбор" ("...") и в стандартном диалоге открытия файла выбрать файл внешней обработки 1С:Предприятия 8.0 (расширение *.epf). После нажатия кнопки "ОК" в диалоге открытия файла в поле "файл" формы элемента справочника будет указано полное имя внешнего отчета.

После нажатия кнопки "Записать" нижней командной панели внешний отчет сохраняется в информационной базе. После этого полное имя внешнего файла заменяется текстом "Отчет загружен в ИБ". То же самое происходит при нажатии кнопки "ОК", при этом форма элемента справочника "Регламентированные отчеты" закрывается.

Для тех элементов справочника, для которых используются внешние отчеты, в форме списка справочника, после наименования отчета добавляется строка "(внешний)".

Работа с регламентированным отчетом, использующим в внешнюю обработку, ничем не отличается от работы с объектом конфигурации, кроме "замедленной реакции" при
открытии и при выгрузке данных отчетности в электронном виде. В качестве признака внешнего отчета в правой части верхней командной панели выводится красная информационная надпись "Внешний отчет".

Пользователь в любой момент может вернуться к использованию внутреннего объекта конфигурации, для этого в форме элемента справочника для требуемого регламентированного отчета необходимо поставить переключатель "Использовать" в положение "объект" и сохранить элемент справочника. Сам внешний отчет не удаляется из информационной базы, поэтому возможно и обратное действие.

Для удаления отчета из информационной базы служит кнопка "Очистить" ("Х") в поле выбора внешнего отчета. После ее нажатия текст "Отчет загружен в ИБ" очищается, это является признаком того, что отчет будет удален из информационной базы. Удаление отчета выполняется при сохранении элемента справочника.

После того, как внешний отчет загружен в информационную базу, сам файл внешнего отчета уже не нужен и его можно удалить. В случае необходимости отчет, загруженный в информационную базу, можно сохранить в файл на диске: для этого в форме элемента справочника "Регламентированные отчеты" предусмотрена
кнопка с изображением дискеты. При ее нажатии будет открыть стандартный диалог сохранения файла. По умолчанию в качестве имени файла предлагается имя соответствующего объекта конфигурации.

В сеансе работы пользователя нельзя использовать более одной версии внешнего отчета. Это значит, что, если пользователь работает с регламентированным отчетом, который является внешним отчетом, и загрузил в справочник "Регламентированные отчеты" другую версию этого внешнего отчета (например, с добавленным автозаполнением), для использования этой версии отчета ему придется закрыть "1С:Предприятие 8.0" и вновь запустить его.
