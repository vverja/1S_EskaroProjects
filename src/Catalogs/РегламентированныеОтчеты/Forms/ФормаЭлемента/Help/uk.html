Довідник "Регламентовані звіти", форма елемента

Кожний елемент довідника "Регламентовані звіти" містить посилання або на один з об'єктів конфігурації, або на зовнішню обробку системи "1С:Підприємство 8.0", що реалізує функції того або іншого регламентованого звіту

При воді нового елемента в довідник необхідно заповнити реквізит "Найменування" - це коротке найменування форми звітності, що буде виводиться в списку регламентованих звітів

Перемикач "Використовувати" служить для того, щоб вказати, яка форма регламентованого звіту буде використовуватися: або об'єкт конфігурації, або зовнішня обробка "1С:Підприємства 8.0".

При виборі перемикача "об'єкт" необхідно в поле введення вказати ім'я об'єкта конфігурації, що буде використовуватися в якості регламентованого звіту. Ім'я об'єкта варто вказувати так, як воно зазначено в конфігураторі.

При виборі перемикача "файл" в поле введення варто вказати файл зовнішньої обробки "1С:Підприємство 8.0". Докладніше про використання зовнішніх звітів див. нижче.

В полі "Опис" можна вказати докладний опис форми звітності, наприклад, повне нейменування форми звітності. Опис, якщо він існує, виводиться для вибраного звіту в нижній частині списку регламентованих звітів

Після натискання кнопки "Записати" зміни в елементі довідника будуть збережені, при цьому:

  • поле для зазначення об'єкта конфігурації буде зроблено недоступним;
  • зовнішній звіт буде завантажений в інформаційну базу, а замість повного імені звіту з'явиться текст "Звіт завантажений в ІБ".

Після натискання кнопки "OK" виконуються ті ж дії, але форма елемента довідника при цьому буде закрита

Використання зовнішніх звітів

У типових конфігураціях для 1С:Підприємства 8.0, що випускаються фірмою "1С", форми регламентованої звітності входять до складу конфігурацій і випускаються разом з релізом конфігурацій

Однак, коли виникає необхідність, як регламентований звіт замість об'єкта конфігурації можна використовувати файл зовнішньої обробки 1С:Підприємства 8.0. Такі файли можуть поширюватися фірмою "1С" або її партнерами в регіонах.

Використання зовнішніх звітів є тимчасовим рішенням, яке варто використовувати тільки до виходу версії конфігурації, в якій будуть
вбудовані необхідні зміни. Тому на використання зовнішніх звітів накладаються певні обмеження:

  • при випуску зовнішнього звіту обмовляється, з якої версії конфігурації він може використовуватися;
  • не гарантується підтримка  зовнішнім звітом попередньої версії конфігурації;
  • при роботі користувача з конфігурацією не можна використовувати більше однієї версії зовнішнього звіту;
  • відкриття форми звіту виконується більш тривалий час, у порівнянні зі звітом реалізованим як об'єкт конфігурації;
  • вивантаження даних звіту в електронному вигляді виконуються набагато повільніше
  • таке  інше

Для використання зовнішнього звіту користувачеві необхідно зазначити, що для даного регламентованого звіту буде використовуватися не внутрішній об'єкт конфігурації, а зовнішній звіт. Для цього в довіднику "Регламентовані звіти" необхідно відкрити на редагування елемент довідника, і в групі "Використовувати" вибрати перемикач "файл". Після цього необхідно вказати, який файл варто використовувати як зовнішній звіт, для цього необхідно нажати кнопку "Вибір" ("...") і в стандартному діалозі відкриття файлу вибрати файл зовнішньої обробки 1С:Підприємства 8.0 (розширення *.epf). Після натискання кнопки "ОК" у діалозі відкриття файлу в поле "файл" форми елемента довідника буде зазначене повне ім'я зовнішнього звіту

Після натискання кнопки "Записати" нижньої командної панелі зовнішній звіт зберігається в інформаційній базі. Після цього повне ім'я зовнішнього файлу заміняється текстом "Звіт завантажений в ІБ". Те ж саме відбувається при натисканні кнопки "ОК", при цьому форма елемента довідника "Регламентовані звіти" закривається

Для тих елементів довідника, для яких використовуються зовнішні звіти, у формі списку довідника, після найменування звіту додається рядок "(зовнішній)".

Робота з регламентованим звітом, що використовує  зовнішню обробку, нічим не відрізняється від роботи з об'єктом конфігурації, крім "уповільненої реакції" при відкритті й при вивантаженні даних звітності в електронному виді. Як ознака зовнішнього звіту в правій частині верхньої командної панелі виводиться червоний інформаційний напис "Зовнішній звіт".

Користувач в будь-який момент може повернутися до використання внутрішнього об'єкта конфігурації, для цього в формі елемента довідника для необхідного регламентованого звіту необхідно поставити перемикач "Використовувати" у положення "об'єкт" і зберегти елемент довідника. Сам зовнішній звіт не видаляється з інформаційної бази, тому можлива й зворотна дія

Для видалення звіту з інформаційної бази служить кнопка "Очистити" ("Х") в полі вибору зовнішнього звіту. Після її натискання текст "Звіт завантажений в ІБ" очищається, це є ознакою того, що звіт буде вилучений з інформаційної бази. Видалення звіту виконується при збереженні елемента довідника

Після того, як зовнішній звіт завантажений в інформаційну базу, сам файл зовнішнього звіту вже не потрібний і його можна видалити. Якщо буде потреба звіт, завантажений в інформаційну базу, можна зберегти у файл на диску: для цього у формі елемента довідника "Регламентовані звіти" передбачена кнопка із зображенням дискети. При її натисканні буде відкритий стандартний діалог збереження файлу. За замовчуванням як ім'я файлу пропонується ім'я відповідного об'єкта конфігурації

У сеансі роботи користувача не можна використовувати більше однієї версії зовнішнього звіту. Це значить, що, якщо користувач працює з регламентованим звітом, що є зовнішнім звітом, і завантажив у довідник "Регламентовані звіти" іншу версію цього зовнішнього звіту (наприклад, з доданим автозаповненням), для використання цієї версії звіту йому доведеться закрити "1С:Підприємство 8.0" і знову запустити його.
