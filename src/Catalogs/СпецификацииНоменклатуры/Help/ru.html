Справочник «Спецификации номенклатуры»

Виды спецификаций

Справочник «Спецификации номенклатуры» предназначен для хранения нормативного состава изделий.

В результате изготовления продукции в общем случае на выходе некоторой операции мы можем получить не один вид продукции, а несколько. То есть спецификации могут иметь множественные выходы. Использование спецификаций на одновременное изготовление нескольких видов продукции устанавливается в реквизите «Вид спецификации».

Возможны следующие виды спецификаций:

  • Сборочная. Это спецификация на изготовление одного вида продукции. В этом случае в группе реквизитов «Выходное изделие» указывается:
    • номенклатура и характеристика выпускаемой продукции, количество и единица измерения количества продукции, на которое задается норматив изготовления. Указание количества позволяет задавать нормативы изготовления, рассчитанные не на единицу продукции, а на заданное количество, так как не всегда нормативы на единицу продукции могут точно описать расход материалов.
    • кратность - эта информация в дальнейшем будет использоваться в системе планирования. Запланированный объем деленный на показатель кратности должен давать целое число.
    • минимальная партия и точка маршрута - указываются для целей планирования, точка маршрута – это совокупность подразделения и рабочего центра, на котором планируется выпуск данной продукции.
    • № операции: Данное значение связывает описываемый выход с одним из узлов технологической (маршрутной) карты (по номеру). Если номер операции указан, то это означает, что выпуск данной номенклатуре в указанном количестве возникает после исполнения операции с таким номером (в технологической карте). Если же номер не указан, это означает, что выпуск производится в момент исполнения производства по спецификации.
  • Полная. Спецификация задается на одновременное изготовление нескольких видов продукции. При этом в форме спецификации показывается табличная часть «Выходные изделия», в которой устанавливается перечень изготавливаемой продукции. Также как и в случае изготовления одного вида изделия для указания списка выходных изделий нужно заполнить данные о номенклатуре, характеристике, количестве продукции, единице измерения количества, кратности, минимальной партии, номере операции и точке маршрута.
  • Доля стоимости. Доля стоимости будет использоваться в дальнейшем следующим образом. Поскольку себестоимость продукции по управленческому учету необходимо рассчитывать для каждой номенклатурной позиции, то все фактически произведенные затраты надо тем или иным способом разнести на объем выпущенной продукции. В документах фактического выпуска продукции, например, в документе «Отчет производства за смену», может быть указано несколько видов использованных материалов и несколько видов получившейся в результате их обработки продукции. Для того чтобы указать, какую часть стоимости материалов отнести на какой вид продукции, в документе указывается доля отнесения затрат. Эта доля должна быть указана для каждого вида продукции в фактическом выпуске. Доли задают базу распределения затрат на виды одновременно произведенной продукции. Согласно этой базе распределяются все затраты, указанные в документах фактического выпуска. По умолчанию, значения долей распределения могут быть заданы в спецификациях на продукцию.
  • Узел. В отличие от номенклатуры номенклатурные узлы не являются материальными сущностями, по ним не ведется количественный складской учет, они могут вообще не существовать в природе как самостоятельные объекты. В номенклатурные узлы выделяются лишь часто используемые при описании спецификаций наборы комплектующих. Поэтому номенклатурный узел описывается элементом справочника «Спецификации» с той лишь разницей, что в такой спецификации отсутствует информация о выходном изделии. Для явного отличия спецификаций на узлы от спецификаций на продукцию в спецификациях на узлы в реквизите «Вид спецификации» выбирается значение «Узел». Указание состава номенклатурного узла аналогично заполнению комплектующих в форме спецификаций. Заметим, что в состав комплектующих узла также могут входить другие узлы.

Версионирование спецификаций

Для каждого вида продукции может быть задан целый ряд спецификаций. Причем спецификации могут задавать действительно разные варианты изготовления продукции, а могут быть версиями одного и того же способа изготовления продукции.

Для того чтобы отличать версии спецификации и основные спецификации используется разграничение по коду. То есть в справочнике «Спецификации номенклатуры» в отдельном реквизите «Код» задается код спецификации, а в другом реквизите «Код версии» - код версий спецификации. Для разных версий одной спецификации значение реквизита «Код» будет одинаковое, а «Код версии» будет разный.

Механизм версионирования позволяет сохранять историю изменения спецификации. Такие изменения могут иметь разные при-чины возникновения: пересмотр норм, использование другого оборудования, замена материалов аналогами и др.

Версии спецификаций можно не вести. Необходимость использования этого механизма указывается в настройках параметров учета. Для того чтобы использовать версии спецификаций, необходимо установить флажок «Использовать версии спецификаций». При установленном флажке становится доступным для ввода реквизит «Код версии», в списке спецификаций код версии также будет виден. Если на предприятии версионирование спецификаций сначала не велось, а потом его решили использовать, то всем спецификациям, имевшимся на тот момент, изначально будет присвоен код версии равный единицы.

Для того чтобы в форме списка спецификаций отобрать все версии одной спецификации можно воспользоваться отбором по реквизиту «Код».

Новую версию спецификации можно ввести, нажав кнопку «Создать версию» на панели инструментов шапки спецификации.

Активность спецификаций

Для каждой спецификации может быть указана ее активность. Активность спецификаций не зависит от использования механизмов версий спецификаций, активность может быть указана как для версии спецификации, так и для основной спецификации. Это выполняется установкой соответствующего флажка «Активная спецификация» в форме спецификации. Активными спецификациями считаются те спецификации (или версии спецификаций), которые реально используются на предприятии для производства продукции. У спецификаций, которые устарели, не используются, признак активности должен быть снят.

Информация об активности спецификаций используется при выборе спецификаций в документы. Для выбора будут доступны только активные спецификации, неактивные спецификации в списке выбора будут не видны.

Для спецификации может вестись учет состояния ее готовности. Оформление данных спецификации может быть отложено, отклонено, находиться в состоянии подготовки и, наконец, утверждено. Состояние спецификации в большей степени носит информативный характер и не участвует в механизмах расчета. Однако признак активности спецификации может быть установлен только для тех спецификаций, которые утверждены. При утверждении спецификации указывается дата утверждения и ответственный за ввод данных спецификации.

Заполнение спецификации

Спецификация имеет несколько разделов для ввода информации. Тематически они разбиты на следующие закладки:

  • Исходные комплектующие. Вводятся данные о материальном составе спецификации.
  • Возвратные отходы. Вводятся данные о нормативном содержании возвратных отходов, возникающих при изготовлении продукции.
  • Параметры выпуска продукции. Указываются значения параметров самой продукции или параметров, описывающих условия изготовления продукции.
  • Документация. Информация о документации, прилагаемой к спецификации.
  • Дополнительно. Дополнительная справочная информация.

Видимость разделов о возвратных отходах, параметрах выпуска, документации управляется в окне настройки, вызываемом по кнопке «Настройка» на командной панели шапки спецификации (раздел называется «Настройка страниц»).

 

Данные в спецификацию можно внести вручную или воспользоваться кнопкой «Заполнить» на командной панели в шапке формы. Возможны следующие стратегии заполнения:

  • Заполнить по спецификации. Будет открыто окно для выбора спецификации. После выбора вся информация содержащаяся в этой спецификации будет скопирована в форму текущей спецификации.
  • Заполнить по фактическим затратам. Будут подобраны все комплектующие, которые за выбранный период были использованы на производство указанной продукции. Такое заполнение имеет смысл выполнять только по тем периодам, по которым уже был выполнен расчет себестоимости продукции. Заполнению подлежат табличные части «Исходные комплектующие» и  «Возвратные отходы».

 

Исходные комплектующие

 

Закладку можно заполнить данными из другой спецификации, для этого на панели инструментов закладки есть кнопка «Заполнить». При ее нажатии будет открыто окно выбора спецификации, откуда копируется состав комплектующих.

 

Заполнение реквизитов:

  • Позиция. Номер комплектующей согласно конструкторской спецификации. Эта информация будет выводиться на печать при формировании печатной формы спецификации по ГОСТ 2.106-96.
  • № операции. Данное значение связывает описываемый вход с одним из узлов технологической (маршрутной) карты (по номеру). Если номер операции указан, то это означает, что потребность в данной номенклатуре в указанном количестве возникает перед началом исполнения операции с таким номером (в технологической карте). Если же номер не указан, это означает, что потребность возникает в самый первый момент исполнения производства по спецификации.
  • Вид норматива. Значениями комплектующих могут быть либо номенклатурные позиции, либо узлы. Причем комплектующие могут быть либо явно указаны пользователем, либо для них задаются только правила выбора (автоподбор номенклатуры, автоподбор характеристики). Способ задания комплектующих в явном виде указывается в реквизите «Вид норматива». Видимость реквизита настраивается в отдельном диалоговом окне, которое открывается при нажатии кнопки «Настройка» в командной панели шапки спецификации. Если предполагается что все комплектующие будут явно указаны пользователем и не будут использоваться узлы, то вид норматива можно не заполнять. Возможны следующие виды нормативов:
    • Номенклатура. Данные о комплектующих выбираются из справочника «Номенклатура». Дополнительно можно указать характеристику комплектующей. Обязательны к заполнению данные о количестве, единицы измерения количества. Помимо количества для целей планирования можно задать кратность.
    • Узел. Данные о комплектующих выбираются из справочника «Спецификации номенклатуры» с видом спецификации «Узел». Данные о характеристики не заполняются.
    • Автоподбор номенклатуры. При выборе этого варианта норматива комплектующая в спецификации не будет выбрана явно из справочника номенклатуры, будет описан сам способ ее выбора. То есть будет установлена зависимость комплектующей от значений характеристик выходного изделия. Поэтому для использования автоподбора учет продукции по характеристикам обязателен. Помимо автоподбора номенклатуры нужно будет установить правила автоподбора для характеристики номенклатуры.
    • Автоподбор характеристики. Прежде чем устанавливать правила автоподбора характеристики, необходимо выбрать комплектующую в реквизите «Номенклатура». Далее для этой комплектующей в отдельной форме устанавливаются правила выбора значения характеристики. Форма открывается при начале выбора реквизита «Характеристика». В этой форме нужно указать перечень свойств, который будет наследоваться комплектующей у готовой продукции. То есть и у готовой продукции и у комплектующей указанные свойства должны быть использованы при определении значений характеристик. Например, при изготовлении продукции цвет комплектующих должен определяться цветом продукции. Понятие «цвет» в этом примере будет являться свойством характеристики и продукции и комплектующих, по значению этого свойства будет выполняться автоподбор характеристики комплектующих.
  • Номенклатура, характеристика номенклатуры – комплектующая, способ ввода данных зависит от выбранного вида норматива.
  • Количество, единица измерения. Количество комплектующей и единица измерения количества. Если вид норматива – «номенклатура», то задается и количество и единица измерения;
    если вид норматива – «узел», то задается только количество;
    если вид норматива – «автоподбор номенклатуры» с вариантом «получить из свойства», то единица измерения в табличной части «Исходные комплектующие» недоступно для выбора, для комплектующих будет использоваться единица измерения остатков;
    если вид норматива – «автоподбор номенклатуры» с вариантом «указывается для свойства», то количество и единица измерения задается явно в форме авто-подбора;
    если вид норматива - «автоподбор характеристики», то количество и единица измерения явно указываются в табличной части.
  • Статья затрат – указать статью затрат, по которой производится учет затрат по комплектующей или всем позициям номенклатурного узла.
  • Мин. партия - Для каждого входа может быть указан размер минимально партии, который определяет минимальное потребляемое количество номенклатуры в указанной единице измерения. Если в ходе расчета нормативной потребности по данной спецификации потребляемое количество окажется меньше минимальной партии, рассчитанное количество будет увеличено до размера минимальной партии.
  • Кратность - эта информация в дальнейшем будет использоваться в системе планирования. Запланированный объем деленный на показатель кратности должен давать целое число
  • Точка маршрута – это совокупность подразделения и рабочего центра, на котором планируется потребление комплектующей.
  • Спецификация. Спецификация изготовления комплектующей. Указывается только в том случае, если вид норматива «номенклатура» или «автоподбор характеристики».
  • Вид воспроизводства. Для целей планирования у комплектующих можно указать вид воспроизводства (покупка, производство, переработка, принятые в переработку). Если выбран вид воспроизводства «производство» или «переработка», то можно указать дополнительно спецификацию изготовления полуфабриката. Реквизиты «вид воспроизводства» и «спецификация» доступны для ввода только если в настройках формы установлен флажок «Использовать вид воспроизводства».
  • Указание норматива. Норматив расхода комплектующих, входящих в состав спецификации, можно задать не только на количество выходных изделий, но и в привязке к количеству основного сырья. В качестве основного сырья можно назначить одну из позиций комплектующих, указанных в табличной части на закладке «Исходные комплектующие». Чтобы установить для комплектующей признак того, что в этой спецификации она является основным сырьем необходимо выбрать соответствующую строку в таблице и нажать кнопку «Основное сырье». Признак основного сырья может быть установлен только для одной строки табличной части. При установке признака для строки, данный признак будет автоматически сниматься у другой строки табличной части, если он был уже установлен ранее. Установка признака основного сырья возможна только если в форме настройки видимости реквизитов включен флажок «Использовать указание норматива». Признак основного сырья нельзя установить для строк с видом норматива «Узел». Для комплектующей, которая назначена основным сырьем нормативы задаются на количество выходных изделий. Для остальных комплектующих можно выбрать один из двух способов задания нормативов в реквизите «Указание норматива»:
    • На количество продукции – расчет потребления комплектующей будет производиться на количество готовой продукции;
    • На количество основного сырья - расчет потребления комплектующей будет производиться на количество основного сырья.
  • Формула. Формула, по которой будет рассчитано количество комплектующей. Ввод формулы выполняется в отдельном диалоговом окне, при задании формулы можно использовать значения параметров выпуска продукции. Формула доступна для ввода, если в настройках формы указан признак «Использовать формулы».
  • Списание комплектующей, Свойство. Реквизит доступен, если в настройке видимости реквизитов формы установлен флаг «Использовать управление списанием». Возможны следующие варианты списания:
    • Всегда. В этом случае считается,  что комплектующая  всегда  используется при выпуске этой продукции и ее распределение на выпущенную продукцию будет производиться согласно способу, указанному непосредственно в документах выпуска. При заполнении табличной части  «Материалы» по спецификации в документах выпуска такая комплектующая будет включена в табличную часть «Материалы» автоматически.
    • В документах распределения. В этом случае также считается,  что комплектующая всегда используется при выпуске этой продукции, но ее распределение на выпущенную продукцию будет производиться только документом «Распределение материалов на выпуск», в течение месяца в отчетах производства за смену расход этой комплектующей нельзя явно указать по каким-либо причинам. При заполнении табличной части «Материалы» по спецификации в документах выпуска такая комплектующая не будет включена автоматически.
    • Задается в свойстве. Данная комплектующая используется на выпуск продукции только при задании в указанном свойстве характеристики выходного изделия значения «Истина». При этом варианте списания в табличной части комплектующих становится доступен реквизит «Свойство». Например, одним из свойств характеристики продукции «кровать» может быть свойство «наличие спинки» типа булево. Это свойство характеристики мы указываем в реквизите «Свойство» для комплектующей «спинка кровати». Тогда если в документах выпуска продукции для кровати будет выбрана характеристика, в которой свойство «наличие спинки» будет иметь значение истины, то комплектующая «спинка кровати» будет подобрана в табличную часть материалов

 

Возвратные отходы

 

Заполнение этой табличной части аналогично заполнению табличной части «Исходные комплектующие» за следующими исключениями:

  • при указании возвратных отходов не используется вид норматива «узел»;
  • в реквизитах «Сумма» и «Сумма (регл)» указывается нормативная стоимость единицы возвратного отхода;
  • не используются такие понятия как «позиция», «кратность», «минимальная партия», «вид воспроизводства», «спецификация».

 

Параметры выпуска продукции

 

На производстве расход комплектующих может зависеть от дополнительных параметров продукции (габариты, вес) или от параметров самого процесса выпуска продукции (влажность). В спецификации можно настроить зависимость расхода комплектующих от параметров выпуска продукции. Для этого в настройках формы спецификации следует установить видимость закладки «Параметры выпуска продукции» и реквизита «Формула». Перечень всех возможных параметров выпуска представляет собой справочник «Виды параметров выпуска продукции». На закладке «Параметры выпуска продукции» следует указать список параметров (реквизит «Вид параметра») и их значений по умолчанию (фактические значения параметров будут заданы непосредственно в документах выпуска продукции). Именно эти параметры в дальнейшем будут участвовать в формулах расчета количества комплектующей. Важно, что параметры должны иметь либо числовой тип, либо тип булево.

 

Табличную часть можно заполнить автоматически данными о параметрах выпуска из другой спецификации. Для этого предназначена кнопка «Заполнить» - «Заполнить по спецификации» на командной панели табличной части.

Для использования одних и тех же комбинаций параметров в разных спецификациях набор параметров и их значений по умолчанию можно записать в шаблон. Для этого используется справочник «Шаблоны параметров». Чтобы использовать шаблон параметров в спецификации на закладке «Параметры выпуска продукции» нужно выбрать кнопку «Заполнить» - «Заполнить из шаблона».

 

Документация

 

Для полноценного формирования печатной формы спецификации по ГОСТ 2.106-96 необходимо выводить сведения о документации. Вводится обозначение документации и наименование. Табличную часть можно заполнить автоматически данными о документации из другой спецификации. Для этого предназначена кнопка «Заполнить» на командной панели табличной части.

 

Дополнительно

 

Реквизиты закладки:

  • Состояние – возможен выбор следующих вариантов:
    • Подготовлен;
    • Утвержден;
    •  Отложен;
    •  Отклонен.
  • Дата утверждения – если состояние спецификации «Утвержден» - то можно указать дату утверждения.
  • Ответственный - сотрудник, ответственный за утверждение спецификации. Указывается при выборе состояния спецификации «Утвержден».
