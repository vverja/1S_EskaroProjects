Настройка прав пользователя

В конфигурации можно задать дополнительные права для пользователей и профилей.

Редактирование дополнительных права выполняется администратором пользователей.

Документы
  • Контролировать превышение объема заказа при отгрузке - при установленном флаге будет производиться контроль отгрузки по заказу покупателя. Документ отгрузки не будет проведен, если в нем будет указано количество товара, превышающее количество товара, выписанного в заказе с учетом корректировок по заказу.
  • Не отпускать товар с ценой ниже определенного типа — выбирается тип цен, ниже уровня которого нельзя изменять цены на товары в документе. Причем цена в документе сравнивается с указанным в типе цен значением с учетом всех скидок, которые были предоставлены покупателю.
    Например. Для пользователя "Петров" задана роль "Менеджер отдела продаж". Для менеджера отдела продаж задан тип цен "Оптовая". Оптовая цена на товар "Ботинки" определена 20 долларов. Покупателю оформили отгрузку товара по цене 20 долларов, но ему предоставили скидку 5%. Таким образом, цена товара в документе получилась 19 долларов. Пользователю "Петрову" запрещено продавать товар по такой цене, следовательно, он не сможет провести такой документ.
  • Печать непроведенных документов — если флаг не установлен, пользователю не будет доступна кнопка "Печать" в документе до тех пор, пока документ не будет проведен.  
  • Разрешить нулевые цены в оптовой торговле - при установке флага пользователю будет разрешено оформлять поступление и отгрузку товара с оптового склада по нулевым ценам.
  • Разрешить нулевые цены в розничной торговле - при установке флага пользователю будет разрешено оформлять поступление и отгрузку товаров с розничного склада и неавтоматизированной торговой точки по нулевым ценам.
  • Разрешить отражать в регламентированном учете - при установке флага пользователю будет разрешено проводить документы по регламентированному учету.
  • Разрешить превышение контролируемых значений по бюджетам - при установке флага пользователю будет разрешено проведение ряда документов при превышении лимитов оборотов по каким-либо комбинациям измерений бюджетирования, установленных документом "Контролируемые значения по бюджетам".
  • Разрешить превышение остатка товаров на складе - при установке этого флага пользователю разрешено отгружать товар даже в том случае, если его нет в наличии на складе.
  • Разрешить превышение остатков товаров организации - при установке флага пользователю разрешено отгружать товар даже в том случае, если товар поступал на другую организацию. 
  • Разрешить превышение свободного остатка денежных средств - пользователю будет разрешено проводить платежные документы на расход денежных средств в том случае, если остаток свободных денежных средств с учетом оформленных заявок на резервирование денежных средств превышен. 
  • Разрешить проведение документа без контроля взаиморасчетов - пользователю будет доступно проведение документа "Реализация товаров и услуг" без контроля взаиморасчетов.
  • Разрешить проведение платежа без заявки — если флаг не установлен, пользователь не сможет провести платежный документ на расход денежных средств без предварительно оформленной заявки на расход средств, которая должна быть указана в платежном документе.
  • Рассчитывать долг при открытии форм — в том случае, если флаг установлен, то при открытии документа будет рассчитан текущий долг по договору или сделке, указанной в документе. Если флаг не установлен, то будет выведена информация — По договору (сделке) долг не рассчитан. Долг будет рассчитан только при нажатии в документе кнопки обновления информации о взаиморасчетах. 
  • Редактирование разрешения превышения лимита отпуска материалов — если флаг установлен, у пользователя будет возможность явно указать в требовании-накладной необходимость списания материалов с превышением установленного лимита.
  • Редактирование таблиц — в том случае, если флаг не установлен, то пользователю не будет доступно редактирование документов и отчетов перед печатью.
  • Редактирование цен и скидок в документах — если флажок не установлен, то пользователю будет недоступно ручное редактирование цен и скидок в табличной части документов, оформляющих продажу товаров. 

Интерфейс кассира
Следующие права используются в документе "Чек ККМ":
  • Разрешить аннулирование чека — если флаг установлен, то пользователю, работающему с интерфейсом кассира, будет доступно аннулирование чека.
  • Разрешить закрытие смены — если флаг установлен, то пользователю, работающему с интерфейсом кассира, будет доступно закрытие кассовой смены.
  • Разрешить назначение ручных скидок — если флаг установлен, то пользователю, работающему с интерфейсом кассира, будет доступно назначение ручных скидок.
  • Разрешить отчет о смене без гашения — если флаг установлен, то пользователю, работающему с интерфейсом кассира, будет доступно формирование отчета о смене без гашения.
  • Разрешить чек возврата — если флаг установлен, то пользователю, работающему с интерфейсом кассира, будет доступно оформление чека на возврат.

Календарь пользователя
  • Возможность просмотра календарей других пользователей — если флаг установлен, то при работе с календарем пользователя будет доступен просмотр календарей других пользователей (в настройке параметров календаря станет возможным редактирование отбора по ответственным).

Рапорт руководителю
Можно назначить доступ к просмотру каждого из показателей отчета "Рапорт руководителя". Если флаг установлен, то пользователю будет доступен показатель из этого отчета, если флаг не установлен, то пользователь не увидит указанного показателя в сформированном отчете.

Справочники
  • Разрешить редактирование КИ в списке — если флаг установлен, пользователь будет иметь возможность редактировать контактную информацию в списке, иначе он сможет редактировать ее только в диалоге.
  • установка флага позволяет пользователю переключать режим редактирования контактной информации (в списке или в диалоге).
