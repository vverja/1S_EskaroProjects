Настройка параметров работы с мобильным приложением

Данные набор настроек предназначен для удобства настройки обмена данными между мобильным приложением, установленным на мобильном устройстве, и информационной базой 1С:Предприятия, а также для дополнительных настроек для работы с модулем обмена для "Агент Плюс". Настройки на форме логически сгруппированы на нескольких вкладках.

Настройка обмена данными включает в себя возможность настройки обмена для 2-х видов приложений, используемых в мобильных устройствах:

1. Агент+ версии 1.5

2. Агент Плюс: Мобильная торговля

Часть настроек для этих приложений пересекаются.

Для переключения настроек, доступных для того или иного приложения используется соответствующие пункты меню "Переключить список настроек", находящегося на верхней панели управления формой. Т.о. настройки, доступные для определенного вида приложения будут отображаться или скрываться.

1. Общие

• Режим выгрузки контрагентов - задается способ выгрузки контрагентов:

1. Из плана посещений – контрагенты будут выгружаться, исходя из документов  "Планирование для агентов" в текущем месяце для конкретного агента;

2.Из договоров агента – выгружаются контрагенты из списка договоров, привязанных для агента. Список договоров задается на форме регистра сведений "Сведения агента". Таким образом, при загрузке документов из МУ будут проставлены, в первую очередь, те договоры, которые привязаны к агенту.

3. Из списка контрагентов для агента – выгружаются контрагента из указанного перечня контрагентов, закрепленных за конкретным агентом. Указание списка контрагентов производится на форме регистра сведений "Сведения агента";

4. Выгружать всех контрагентов – выгружаются все контрагенты-покупатели из справочника «Контрагенты».

5. Из плана и договоров – комбинированный способ выгрузки контрагентов. При выгрузке будут учтены контрагента, которые указаный в плане посещений и те, которые указаны в списке договоров в регистре сведений "Сведения агента". При выгрузке взаиморасчетов в этом случае будут учтены только долги по указанному списку договоров. Но, если в спике договоров нет контрагента, которые есть в плане посещений, то выгрузка долгов будет произведена по всем договорам.   

• Режим выгрузки номенклатуры - выбирается режим выгрузки номенклатуры в МУ:

1. Не учитывать разрез – остатки товаров будут выгружаться без учета характеристик и серий;

2. В разрезе характеристик номенклатуры – выгрузка номенклатуры с учетом характеристик;

3. В разрезе серий номенклатуры – выгрузка номенклатуры с учетом серий (данная настройка недоступна для приложения "Агент Плюс: Мобильная торговля").

• Режим выгрузки долгов - способ выгрузки взаиморасчетов:

1. По взаиморасчетам с контрагентами. Долги в этом случае будут выгружаться в разрезе договоров контрагентов;

2. В разрезе документов расчетов с контрагентами. При этом долги в МУ будут выгружаться в разрезе документов расчетов с контрагентами (в договоре контрагента должен быть указан признак ведения взаиморасчетов по документам расчетов).

• Режим загрузки заказов - указывается способ загрузки заказов:

1. Не создавать документ реализации. При загрузке заявок из МУ будут создаваться документы "Заказ покупателя" в ЦБД. 

2. Создавать на основании заявки (метод "Ввод на основании"). При загрузке заявок из МУ будет создаваться документ "Заказ покупателя", а также на основании него  будет создан документ "Реализация товаров и услуг" с использованием стандартного механизма "ввода на основании", используемого в конфигурации.

3. Перенос содержимого заявки в реализацию. В отличие от предыдущего способа, в котором используются стандартный механизм "заполнения на основании" конфигурации, при этом способе содержимое заявки полностью переносится в документ "Реализация товаров и услуг" без всяких дополнительных проверок.

• Режим загрузки документов поступления - выбирается способ загрузки документа «Поступления» из МУ:

1. Как документ «Возврат товаров от покупателя» – в ЦБД будет создан документ «Возврат товаров от покупателя»;

2. Как документ «Поступление товаров и услуг» – в ЦБД будет создан документ «Поступление товаров и услуг».

• Режим использования торговых точек - выбирается один из двух типов: либо «Использовать», т.е. с использованием торговых точек, подчиненных контрагенту, либо «Не использовать» - если организация работает с клиентами без использования торговых точек (при этом никаких данных, связанных с использованием торговых точкам, при обмене данными выгружаться и загружаться не будет).

• Количество выгружаемых единиц измерения  - указывается количество единиц измерения номенклатуры, которое возможно выгрузить в МУ. Минимальное значение 3, максимальное 10.

• Использовать единицы измерения - предназначенная для указания использования единиц измерения в табличной части документов в МУ.

• Использовать персональные цены - если выбран этот флажок, то для клиентов будут выгружаться персональные цены при условии, если в регистре сведений "Сведения контрагентов" для соответствующего клиента выбран флажок "Использовать персональные цены".

• Резервирование заказов - выполнять резервирование номенклатуры из загруженных заказов.

• Использовать договоры - признак использования договоров в МУ. Если установлена данная константа, то в МУвключается режим работы с договорами (на форме документов появляется поле для выбора договора). Если используется режим выгрузки контрагентов по договорам (см. описание константы "Режим выгрузки контрагентов"), то в МУ будут выгружены только договоры из указанного списка, в противном случае будут выгружаться все договоры, принадлежащие контрагенту. Соответственно, при загрузке документов с МУ, в реквизит "Договор" в документах ЦБД будет проставлен тот договор, который был указан в МУ.

 

2. Дополнительные настройки

• Срок выгрузки перемещений - количество дней, за которое выгружаются документы перемещения;

• Интервал планирования - выбирается вариант составления документа "Планирование агентов";

• Интервал подтверждений - глубина выгрузки подтверждений (в днях) на созданные в МУ документы;

• Название БД - название базы данных в карманном компьютере;

• Каталог дополнительной информации - стандартная константа конфигурации «Управление торговлей 10.3», выведена на форму констант для удобства заполнения данных. Содержит каталог, используемый для хранения промежуточных и временных файлов, в частности, файлов выгрузки данных для МУ;

• Папка картинок товаров в МУ- директория, в которой хранятся картинки товаров.

• Папка фото к документам в МУ- директория, в которой хранятся фотографии к документам.

• Группа новых клиентов - группа справочника «Контрагенты», в которую загружаются созданные в МУ контрагенты;

• Использовать планы продаж. Если установлен флажок, то производится выгрузка план-фактных данных по продажам. Если указан данный флажок, то становится доступной константа:

• Режим планирования продаж - режим плана продаж, используемый по умолчанию.

• Включить автозаполнение регистров сведений - если установлена данная константа, то в процессе выгрузки данных автоматически будут заполнены регистры сведений «Сведения о контрагентах» и «Сведения организация» объектами, которые выгружаются в МУ. Этот механизм предназначен для упрощения заполнения данных регистров сведений, а также для возможности быстрой настройки дополнительных параметров контрагентов и организаций.

• Использовать запрет продаж в МУ - использовать запрет оформления документов продаж в МУ для клиентов при нарушении условий основного договора.

 

3. Типы цен

На данной вкладке указывается список типов цен, которые должны быть выгружены в МУ. Предусмотрено заполнение списка из справочника "Типы цен номенклатуры".

 

4. Константы, используемые для настройки обмена данными

• Способ обмена по умолчанию - способ обмена, используемый по умолчанию при открытии обработки "Обмен данными с мобильными устройствами"

• Пользователь для фонового режима - в данную  константу следует выбрать пользователя, от имени которого будет выполняться обмен данными в фоновом режиме (т.е. на стороне сервера 1С:Предприятия). Заполнение данной константы имеет смысл только для работы информационной базы в режиме клиент-сервер. При использовании файл-серверного режима пользователь задается в основной форме констант «Настройка параметров учета» на вкладке «Обмен данными».

• Крипто-пароль - пароль для дополнительного шифрования файлов;

• Шифровать данные - включает / выключает шифрование данных.

 

Константы для настройки обмена через FTP-cервер• FTP-Сервер - адрес FTP-Сервера;

• FTP-Пользователь - имя FTP-Пользователя;

• FTP-Пароль - FTP-Пароль.

 

Константы для настройки обмена через Агент+ Сервер

• Адрес сервера - сетевой адрес компьютера и порт (порт пишется через двоеточие после адреса сервера), на котором запущен Агент+ Сервер.»

• Папка обмена - каталог обмена данными через «Агент+ сервер».

 

5. Настройки для GPS

• Папка хранения файлов с картами - директория, в которой хранятся карты для отображения маршрута агента;

• Основная карта - основная карта для отображения маршрутов агентов (агента). Используется как карта по умолчанию, которая выбирается в отчетах "Карта маршрута агента" и "Контроль передвижений агентов".

• Приложение для просмотра карт - выбирается тип используемого приложения для просмотра карт. Может быть выбрано либо использование стандартной утилиты компоненты APPlus.dll, либо использование внешнего приложения Map9 Viewer.

• Путь к файлу запуска Map9 Viewer - указывается путь к файлу запуска приложения Map9 Viewer (необходима при формировании отчета "Контроль передвижений агентов").

• Режим загрузки треков - выбирается способ загрузки треков перемещения агентов из МУ: 

     - Загружать каждую точку трека в ЦБД;

     - Загружать файл треков в ЦБД целиком (только при данном способе загрузки важно указание константы, которая описана ниже).

• Папка треков перемещений - указывается каталог, предназначенный для хранения файлов с треками перемещений агентов, которые загружаются из МУ. 

• Погрешность при расчете факта посещения точки - погрешность в метрах, которая влияет на расчет факта посещения торговой точки/клиента при формировании отчета "Карта маршрута агента" в виде расшифровки.
