Обработка «График заказов»

Обработка предназначена для анализа планируемых сроков выполнения заказов на производство и заказов покупателей, размещенных в заказах на производство.

В график попадают следующие заказы:

  • Заказы на производство, в которых заполнены реквизиты "Дата запуска" и "Дата исполнения"
  • Заказы покупателей, которые размещены в заказах на производство.
  • Заказы поставщикам, в которых размещены потребности заказов на производство 
  • Внутренние заказы, в которых размещены потребности заказов на производство либо которые размещены в заказах на производство

    Непроведенные документы в график не попадают.

    Период выполнения заказа на производство определяется на основе реквизитов документа "Заказ на производство": "Дата запуска" и "Дата исполнения".

    Период выполнения заказа поставщику, внутреннего заказа и заказа покупателя определяются исходя из реквизитов "Дата поступления" и "Дата отгрузки" соответственно. Для заказов поставщику, внутренних заказов и заказов покупателей период выполнения всегда принимается 1 день.

    Связь между заказами отображается в виде стрелок. Связи устанавливаются на основании сведений о размещении заказов. Обязательным является указание периода формирования графика. В график попадут заказы, период выполнения которых попадает в период формирования графика.

    Работа с обработкой

    Изменение графика

    Изменение графика разрешено, если в форме настройки (кнопка "Настройка") установлен соответствующий признак. В этом случае с помощью перетаскивания можно оперативно менять периоды выполнения заказов:

    • Переносить дату отгрузки / дату поступления для заказов покупателей, внутренних заказов и заказов поставщикам
    • Переносить период выполнения заказа на производство (перетаскивание отрезка диаграммы)
    • Менять длительность выполнения заказа на производство (перетаскивания начала либо окончания отрезка диаграммы)

    Измененные заказы выделяются на графике жирным шрифтом.
    Внесенные изменения будут сохранены в информационной базе после нажатия на кнопку "Сохранить график". На этапе сохранения изменений можно отказаться от изменений по некоторым заказам (отключить признак в списке измененных заказов).

    Если закрыть форму обработки либо переформировать график без нажатия на кнопку "Сохранить график", внесенные изменения не сохранятся. Отменить несохраненные изменения можно с помощью соответствующей кнопки на командной панели отчета.

    Просмотр сведений о заказах, отображенных в обработке

    С помощью двойного щелчка на представлении заказа можно выполнить следующие действия:

    • Открыть форму заказа
    • Сформировать структуру подчиненности данного заказа
    • Сформировать отчет "Анализ заказа"

    Конкретное действие выбирается из открывающегося списка.

    Выделение цветом

    • Заказы на диаграмме выделяются цветом:
      • исполненные в срок - голубым
      • исполненные с опозданием - фиолетовым
      • просроченные - красным
      • все остальные - зеленым
    • Связи между заказами на диаграмме
      • В случае несоответствия даты выполнения и даты запуска связанных между собой заказов связь между данными заказами отображается стрелкой красного цвета
      • Связь, выходящая из невыполненного просроченного заказа, отображается стрелкой красного цвета
      • В остальных случаях связи отображаются стрелками синего цвета 
    • Дни на диаграмме отличаются фоном
      • выходные и праздничные дни выделяются красным
      • текущая дата выделяется желтым

    Диагностика

    С помощью кнопки "Диагностика" командной панели можно вывести отчет о проблемах с графиком заказов:

    • Просроченные заказы
    • Заказы выполненные с опозданием
    • Потребности заказа не будут обеспечены к дате запуска

    Диагностику можно выполнить только после формирования графика. Несохраненные изменения графика заказов в диагностике не учитываются.

    Из полученного отчета можно открывать формы заказов.

    Настройка обработки

    Настройка выполняется в отдельной форме, которая открывается с помощью кнопки "Настройка" командной панели.

    Разрешены изменения

    Разрешить изменения графика можно установкой соответствующего признаком на закладке "Настройка". Изменения графика выполняются с помощью перетаскивания: таким образом можно изменять периоды выполнения заказов. Подробнее см. выше пункт "Изменения графика".

    Обновлять диаграмму после изменения заказов

    При установке данного признака данные графика будут автоматически обновляться, если из формы были открыты и отредактированы заказы. Если данный признак не установлен, для обновления графика потребуется использовать кнопку "Сформировать график".

    Отображение рабочих и выходных дней

    Настройка для отображения рабочих и выходных дней. Могут использоваться следующие варианты:

    • По регламентированному производственному календарю. В случае, если регламентированный производственный календарь не заполнен, в качестве рабочих принимаются дни с понедельника по пятницу, в качестве выходных - субботы и воскресенья.
    • По графику. При выборе данного варианта потребуется указать график

    Отбор

    Отбор можно установить для отображаемых на графике заказов на производство. Установка отбора для заказов покупателей и заказов поставщикам не предусмотрена

    Настройка представления заказов

    Обработка позволяет настраивать список полей, которые будут участвовать в формировании представления заказов. Для каждого вида заказа устанавливаются свои настройки.
    Например, для заказов на производство можно настроить представление в виде "Номер + Дата + Подразделение".
    По умолчанию представлением заказов является полное представление документа.
