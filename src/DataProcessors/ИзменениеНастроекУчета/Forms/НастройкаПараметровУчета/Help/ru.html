Настройка параметров учета

Настройка параметров учета вызывается из меню "Сервис - Настройка учета" выбором пункта "Настройка параметров учета". Ее назначением является ввод параметров, по которым будет вестись учет в компании.

Производство
  • Тип цен плановой себестоимости номенклатуры - если установлен порядок формирования стоимости МПЗ и выпущенной продукции (работ, услуг) в течение отчетного месяца по плановым ценам, то учет номенклатуры будет вестись по ценам этого типа из регистра сведений "Цены номенклатуры".
  • Использовать только сборочные спецификации - при установке флага можно будет создавать спецификации номенклатуры только с видом "Сборочная". Если флаг не установлен, могут использоваться спецификации с видом "Сборочная", "Полная", "Узел".
  • Использовать версии спецификаций - при установке флага в конфигурации будет доступно указание версии в спецификации номенклатуры.
  • Использовать лимиты отпуска материалов - при установке флага в конфигурации будет разрешена работа с механизмом лимитно-заборных карт.
  • Использовать наработку - при установке флага в документе "Отчет производства за смену" можно будет указывать вид выпуска "Наработка".

Затраты и себестоимость
  • Способ учета затрат - выбирается режим учета себестоимости и затрат.
    • Использовать расширенную аналитику учета затрат (РАУЗ) - выбор этого режима рекомендуется для предприятий со сложным учетом затрат. Режим расширенной аналитики объединяет партионный учет и учет затрат в производстве, а также обслуживает все счета учета материально-производственных запасов (включая продукцию) и затрат. При ведении учета нет зависимости от хронологического порядка ввода документов. Также будет возможен более детальный учет, позволяющий получать аналитические данные по движениям номенклатуры и затрат.

Для изменения установленного режима использования расширенной аналитики необходимо выполнить следующее:

  • снять флаг "Использовать расширенную аналитику учета затрат"
  • удалить документы ввода остатков по расширенной аналитике
    • либо вручную
    • либо с помощью мастера, который открывается при установке флага "Использовать расширенную аналитику учета затрат"
  • установить флаг "Использовать расширенную аналитику учета затрат" и выбрать новый режим
  • после изменения режима ведения учета перепровести все документы, относящиеся к учету МПЗ и затрат, начиная с даты перехода на расширенную аналитику
    • Использовать партионный учет - при установке флага последовательность расчета себестоимости будет включать в себя большее количество этапов, чем при использовании РАУЗ, так как для учета стоимости МПЗ и для учета затрат в производстве будут использоваться разные механизмы с отдельными наборами регистров.
      • Списывать партии при отражении документов (в управленческом учете, в регламентированном учете) - при установке флага движения по соответствующим регистрам партионного учета и проводки по себестоимости (для регламентированного учета) будут формироваться непосредственно при проведении документов. Также будут формироваться движения по регистрам партионного учета НДС. Это позволяет оперативно выявлять ошибки списания партий и получать себестоимость списанных партий на момент проведения документов. Если флаг не установлен,  то проведение по партиям можно осуществить с помощью специальной обработки "Проведение по партиям". Параметр устанавливается отдельно для управленческого и регламентированного учета.

При интенсивном документообороте запрещается устанавливать флаги "Списывать партии при отражении документов", иначе это приведет к снижению показателей производительности и параллельности системы.

      • Последовательность списания партий - устанавливается очередность списания партий собственных (купленных) и принятых на реализацию товаров.
      • Списывать партии расходным ордером - если флаг установлен, то при реализации товаров по ордеру списание партий товаров будет производиться документом "Расходный ордер на товары". Если флаг не установлен, то партии товаров будут списываться при проведении документов реализации.
  • Вести учет себестоимости МПЗ по складам (в управленческом учете, в регламентированном учете) - при установке флага по каждому складу будет вестись суммовой и партионный учет, а применение одного из способов оценки стоимости МПЗ при выбытии (ФИФО, ЛИФО, по средней стоимости) будет осуществляться также по каждому складу отдельно. Такой способ ведения учета имеет смысл только в случае, если разные склады находятся в местах с существенно различными условиями ведения хозяйственной деятельности. Если флаг не установлен, то партионный и суммовой учет МПЗ, а также способ оценки стоимости МПЗ при выбытии применяются вне зависимости от конкретного склада. Параметр устанавливается отдельно для управленческого и регламентированного учета.

При изменении установленного ранее порядка ведения партионного учета по складам необходимо выполнить ряд дополнительных действий. Например, 15 ноября требуется изменить настройку, начиная с 01 ноября. Для этого следует:

  1. проверить выполнение всех регламентных операций по закрытию месяца за октябрь, включая восстановление последовательности партионного учета
  2. оформить с помощью документа "Списание товаров" списание всех товарно-материальных ценностей, находящихся на складах, на 31 октября
  3. установить/снять флаг "Вести партионный учет по складам" по соответствующему виду учета
  4. оприходовать на дату 31 октября товарно-материальные ценности, списанные в соответствии с п. 2
  5. установить дату запрета изменения данных 31 октября
  6. перепровести все документы, начиная с 01 ноября
  • Вести управленческий учет себестоимости по организациям - устанавливается порядок ведения управленческого партионного учета по организациям (по умолчанию ведется по предприятию в целом):
    • Не ведется - управленческий партионный учет по организации не ведется;
    • По компании в целом - управленческий партионный учет ведется по предприятию в целом (без учета организации);
    • По головной организации - управленческий партионный учет ведется в целом по организациям, объединенным общей головной компанией;
    • По текущей организации - управленческий партионный учет ведется отдельно для данной организации.

Учет товаров
  • Использовать характеристики номенклатуры, Использовать серии номенклатуры, Использовать серийные номера - при установке флагов ведется учет номенклатуры по характеристикам, сериям, серийным номерам. Если флаги не установлены, то в документах и справочниках не будут показываться соответствующие поля для учета серий и характеристик.
  • Вести учет возвратной тары - при установке флага в конфигурации становится возможным оформление операций учета возвратной тары.
  • Указывать склады в табличной части документов - значение реквизита определяет возможность указания нескольких складов в табличной части документов поступления и реализации.

Печать, единицы измерения

  • Дополнительная колонка печатных форм документов - информация в данном поле определяет ту дополнительную информацию, которая выводится в печатных формах документов и в отчетах.

    • Код - в печатных формах документов и в отчетах будет дополнительно выводиться код, заданный для позиции номенклатуры. 
    • Артикул - в печатных формах документов и в отчетах будет дополнительно выводиться артикул, заданный для позиции номенклатуры. 
    • Не выводить - дополнительной информации выводиться не будет.

  • Единица измерения веса -  в данном поле необходимо задать краткое наименование единицы веса, используемой для измерения веса позиции номенклатуры в конфигурации.

  • Единица измерения объема - в данном поле необходимо задать краткое наименование единицы объема, используемой для измерения объема позиции номенклатуры в конфигурации.

Взаиморасчеты и денежные средства
  • Использовать оплату платежными картами - при установке флага в конфигурации разрешается использовать оплату от покупателей платежными картами.
  • Использовать оплату банковскими кредитами - при установке флага в конфигурации разрешается использовать оплату от покупателей банковскими кредитами.
  • Способ контроля дней задолженности - выбирается способ контроля дней задолженности, который будет использоваться  при оперативном контроле взаиморасчетов с контрагентом, если необходимость такого контроля определена в договоре с контрагентом.
  • Если у документа совпадают даты регистрации и оплаты, то проводить его - определяет порядок проведения банковских платежных документов при совпадении даты регистрации документа (реквизит "от") и даты его оплаты (реквизит "Оплачено").

Движения регистров взаиморасчетов и проводки по счетам 36, 63 формируются днем, указанным в реквизите "Оплачено". Время движений и соответственно момент времени, на который платежный документ определяет состояние расчетов, зависит от совпадения дня даты документа с днем даты "Оплачено" и от учетной политики.
Если день даты платежного документа не совпадает с днем даты "Оплачено", то движения будут делаться концом дня, указанного в реквизите "Оплачено" (23:59:59), следовательно, расчет задолженности для определения аванса будет производиться на дату "Оплачено" и время 23:59:59 (включительно).
Если день даты платежного документа совпадает с днем даты "Оплачено", то время движений по документу зависит от варианта учетной политики:

    • По времени регистрации документа - устанавливается для корректного отражения взаиморасчетов с контрагентами (аванс или оплата).
    • В конце дня даты регистрации документа - может устанавливаться при неоперативной регистрации хозяйственных операций.
  • Вести аналитический учет денежных средств по обособленным подразделениям - при установке флага в конфигурации на счетах учета денежных средств в кассе будет использоваться субконто "Обособленные подразделения".

Заказы
  • Использовать внутренние заказы - при установке флага в конфигурации будет доступно использование внутренних заказов (документ "Внутренний заказ").
  • Стратегия авторезервирования по заказам - выбирается стратегия авторезервирования по заказам, которая будет подставляться в документах по умолчанию при использовании сервисов автоматического резервирования и размещения. Возможные значения: "Сначала на складах потом в заказах поставщикам", "Сначала в заказах поставщикам потом на складах".
  • Указывать серии при резервировании - при установке флага становится возможным выполнять резервирование товаров на складах с учетом серий. Резервирование с учетом серий может выполняться только при резервировании под заказы покупателей, в которых указан договор контрагента с установленным признаком "Обособленный учет товаров по заказам покупателя".
  • Указывать заказы в табличной части документов - значение реквизита определяет возможность указания заказов в табличной части документов поступления и реализации.
  • Учитывать возвраты товаров покупателем - при установке данного флага документ "Возврат товаров от покупателя" увеличивает объем заказанных товаров по заказам, указанным в документе.

Заказы на производство
  • Использовать заказы на производство - при установке флага в конфигурации будет доступно использование заказов на производство (документ "Заказ на производство").
  • Вести учет потребностей заказов на производство в материалах и полуфабрикатах - при установке флага ведется учет потребностей заказов на производство в материалах и полуфабрикатах. Если флаг не установлен, а раздел "Материалы" в документе "Заказ на производство" заполнен, то данная информация является справочной и не участвует в анализе удовлетворения потребности заказа. Признак доступен для изменения, если установлен признак "Использовать заказы на производство".
  • Способ закрытия потребностей заказов на производство:

    • Явно - закрытие потребностей происходит с помощью документа "Корректировка заказа на производство". Также потребности заказа на производство полностью закрываются при проведении документов "Отчет производства за смену", "Комплектация номенклатуры" и "Акт об оказании производственных услуг", если по заказу на производство выпущена вся продукция.
    • Автоматически при распределении - закрытие потребностей происходит при распределении материалов на выпуск продукции, а также при оформлении выпуска продукции с помощью документа "Комплектация номенклатуры".

Планирование
  • Периодичность доступности ключевых ресурсов предприятия - определяет период, в рамках которого задаются показатели доступности ключевых ресурсов. Используется для оценки исполнимости планов.
  • Вести планирование производства по сменам - при установке флага используется функционал посменного планирования, в спецификациях и в заданиях на производство становятся видимы реквизиты для ввода информации о технологических операциях.

Проекты
  • Вести учет по проектам - при установке флага используется дополнительная детализация продаж, закупок, движений денежных средств, затрат и планирования в разрезе проектов.
  • Использовать виды распределения по проектам - при установке флага появляется возможность указать, нужно отнести затраты на конкретный проект или распределить их по проектам: в документах и справочниках будет возможен выбор вида распределения по проектам. Если флаг не установлен, можно выбрать только проект. Виды распределения по проектам содержатся в справочнике "Виды распределения по проектам".
  • Вести учет затрат по проектам - при установке флага ведется учет косвенных затрат по проектам в управленческом учете. Такая возможность предусмотрена для затрат с характером затрат "Издержки обращения", "Коммерческие расходы", "Общепроизводственные расходы", "Общехозяйственные расходы", "Прочие".
  • Указывать проекты в табличной части документов - при установке флага в документах, которыми можно отражать затраты, будет доступно указание проектов в табличной части.

Валюты
  • Валюта регламентированного учета — базовая валюта, курс которой всегда равен 1 (для Украины — гривна). Курсы всех других валют выражаются в единицах валюты бухгалтерского учета.
  • Валюта управленческого учета — валюта, используемая для оценки данных в управленческом учете.
  • Валюта международного учета — валюта, используемая для ведения учета по МСФО (функциональная валюта).
