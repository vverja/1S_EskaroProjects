Обработка "МСФО Консолидация"

Обработка «Консолидация по МСФО» предназначена для автоматизированного составления консолидированной отчетности группы  по международным стандартам и построена по принципу мастера – пользователю для получения результата нужно выполнить определенную последовательность действий.

Обработка предоставляет две возможности подготовки консолидированной отчетности:

  • По данным международного учета, имеющимся в информационной базе
  • По данным индивидуальной отчетности организаций, входящих в группу, имеющейся в информационной базе

Выбор одной из этих возможностей, так же как и задание отчетного периода (года) осуществляется на первых этапах использования обработки.

Ограничение: пока не предусмотрено составление промежуточной отчетности (только годовая).

В дальнейшем в случае выбора способа формирования консолидированной отчетности по данным учета выполняются следующие действия:

  • заполнение списка организаций, чьи учетные данные будут использоваться для построения консолидированной отчетности (по кнопке "Заполнить" будет предложен список всех собственных организаций, данные о которых имеются в информационной базе, затем этот список можно скорректировать);
  • ввод данных об остатках нереализованной прибыли от внутригрупповых операций в активах для ее исключения при консолидации;
  • попытка автоматического распознания внутригрупповых операций и объектов учета и формирования элиминирующих проводок;
  • заполнение консолидированной отчетности.

Элиминирующие проводки записываются в международный учет без указания организации, таким образом, данные индивидуального учета каждой организации не нарушаются.

Если по результатам анализа окажется, что обработкой сформированы не все записи, которые необходимы для элиминирования статей (исключения внутригрупповых операций), выполнение обработки можно прервать (даже не закрывая окно обработки, если необходимо), ввести необходимые записи (например, при помощи документа «Операция (международная)» без указания организации), и затем вернуться к формированию консолидированного отчета.

Внимание! При повторном запуске обработки (после закрытия ее окна) все будет сформировано заново (это не документ, где все данные сохраняются), в том числе элиминирующие проводки, которые система сможет определить автоматически. При этом аналогичные проводки, сформированные обработкой при прошлом запуске (запусках), автоматически удалены не будут. Поэтому при необходимости их надо будет удалить перед формированием отчетности, что несложно сделать, отобрав документы "Операция (международная)" с неуказанной организацией.

В случае выбора способа формирования консолидированной отчетности по данным индивидуальной отчетности выполняются следующие действия:

  • поиск индивидуальной отчетности в информационной базе и заполнение списка организаций, по которым нужно получить консолидированную отчетность;
  • обработка примечаний к индивидуальной отчетности каждой организации, касающихся внутригрупповых операций, взаимных инвестиций и т.д.
  • заполнение консолидированной отчетности

Перед запуском обработки должны быть подготовлены и проверены все исходные данные:

  • справочник "Организации" должен выключать в себя все организации, которые входят в группу (независимо от того, ведется ли учет операций этих организаций в данной информационной базе или не ведется);
  • регистр сведений «Контрагенты, связанные с организациями» (должен содержать сведения обо всех контрагентах, которые являются связанными сторонами по отношению к организациям, включенным в справочник «Организации», и типы связей - какой контрагент является дочерней компанией организации, какой – материнской, и т.п.);
  • регистр сведений "Собственные контрагенты" (должен быть также заполнен);
  • данные международного учета каждой организации, которую предполагается включить в список для консолидированной отчетности, если имеется намерение составлять консолидированную отчетность по данным учета;
  • индивидуальная отчетность тех организаций, которую предполагается консолидировать (если предполагается консолидация по отчетности), включая примечание по внутригрупповым операциям.
