Настройки отчета

Форма настройек отчета вызывается из форм регламентированных отчетов, состоящих из множества разделов (например, это почти все налоговые
декларации).

Форма предназначена для:

  • управления автоматической нумерацией страниц отчета;
  • управления порядком заполнения тех полей отчета, значения которых должны выбираться из заранее определенного набора значений;
  • управления автоматическим сохранением отчета;
  • управления показом и печатью разделов отчета.

«Отключить счетчик страниц» - если флажок установлен, номера страниц отчета не будут вычисляться автоматически при формировании печатной формы.

«Отключить выбор значений из списка» - установка этого флажка позволяет отключить вывод на экран списка допустимых значений полей налоговых деклараций, для которых предусмотрены такие списка, и указать значение вручную.

«Автосохранение каждые … минут» - если флажок установлен, при редактировании регламентированного отчета он будет автоматически сохраняться с указанным интервалом времени. Для новых отчетов автоматическое сохранение включается после того, как отчет будет записан в журнале отчетов. Также автоматическое сохранение не будет производиться в следующих случаях:

- если в процессе редактирования отчета был изменен вид документа и/или номер корректировки и после этого данные не были сохранены;

- если в процессе редактирования отчета был изменен код ИФНС (если такой реквизит присутствует на форме), в которую представляется отчет, и после этого данные не были сохранены.

«Наносить двухмерный штрихкод» - если флажок установлен, на листы налоговой декларации при печати будет наноситься двухмерный штрихкод (см. раздел «Технология двухмерного штрихкодирования налоговых деклараций» ниже). Флажок доступен для установки в том случае, если на компьютере пользователя установлена специализированная библиотека программного генератора компактного двухмерного штрихкода Aztec. В случае, если такая библиотека не установлена, флажок сбрасывается автоматически, и печать деклараии выполняется в обычном режиме - без двухмерного штрихкода.

«Проверять соотношения показателей при печати и выгрузке» - если флажок установлен, то перед печатью и выгрузкой будет автоматически запускаться проверка показателей отчета.

Список разделов отчета - в списке можно указать, какие из разделов отчета следует показывать на экране и выводить на печать. Разделы для показа и вывода на печать можно указывать независимо: например, раздел можно заполнить и спрятать, но при печати он будет обязательно напечатан.

Технология двухмерного штрихкодирования налоговых деклараций

Технология двухмерного штрихкодирования налоговых деклараций, включенная в типовые конфигурации для «1С:Предприятия 8.0»,  носит экспериментальный характер.

Суть данной технологии заключается в следующем.

При печати налоговой декларации данные декларации выгружаются в электронном виде в формате, утвержденном соответствующим приказом ФНС РФ, и наносятся на печатную форму в виде блоков компактного двухмерного штрихкода Aztec. Формирование блоков штрихкода обеспечивается специализированной библиотекой, разработанной и распространяемой ФНС РФ. Подготовка налоговых деклараций с нанесенным двухмерным штрихкодом возможна только при использовании специально разработанного программного обеспечения – в частности, экономических программ  фирмы «1С».

Для применения данной технологии пользователям необходимо установить на компьютере специализированную библиотеку генерации двухмерного штрихкода. Библиотека в виде инсталляционного комплекта расположена на сайте ФГУП ГНИВЦ ФНС РФ по адресу http://217.106.225.43/gnivcsoft/techno/setup_BarCodeLib.exe.  Установка библиотеки из инсталляционного комплекта выполняется обычным порядком и не должна вызывать трудностей.

Библиотека генерации двухмерного штрихкода, порядок нанесения двухмерного штрихкода на налоговые декларации и примеры декларации с нанесенным двухмерным штрихкодом опубликованы на сайте ФГУП ГНИВЦ ФНС РФ http://www.gnivc.ru/techno/211.htm

Двухмерный штрихкод, нанесенный на налоговые декларации, обеспечивает их ускоренную обработку в налоговой инспекции и отсутствие ошибок при вводе в системы камеральной проверки. Для возможности приема деклараций с двухмерным штрихкодом налоговый орган должен быть оснащен специализированными сканерами и соответствующим программным обеспечением.
