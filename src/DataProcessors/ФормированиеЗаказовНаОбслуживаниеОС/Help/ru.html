Обработка «Формирование заказов на обслуживание ОС»

Обработка используется в подсистеме управления обслуживанием основных средств, предназначена для определения ОС, требующих ремонта и формирования заказов на обслуживание ОС.

Перед открытием основной формы обработки открывается форма настройки заполнения основных средств.

Для быстрого заполнения табличной части однотипными объектами основных средств, имеющими одинаковые наименования, нужно ввести в табличную часть хотя бы один такой объект. Затем следует воспользоваться кнопкой Заполнить в командной панели табличной части. При выборе в выпадающем меню пункта По наименованию табличная часть будет заполнена объектами основных средств, имеющими такое же наименование, как у введенного первоначально.

Пользователь может установить фильтры на основные средства (группы ОС), которые будут анализироваться и услуги по обслуживанию ОС. Форма настройки так же может быть открыта из основной формы обработки, кнопкой «Заполнить» верхней командной панели формы. Если необходимо проанализировать все ОС и все услуги на обслуживание, то заполнять табличные поля отбора не следует. При нажатии на кнопку «Выполнить» производится анализ необходимости обслуживания ОС исходя из нормативов обслуживания ОС, состава ОС для обслуживания в нормативах, даты последнего обслуживания ОС и периодичности обслуживания, значения параметра выработки и значения параметра выработки по нормативу. ОС для которых необходимо проводить обслуживание заполняются в табличное поле обработки.

Основное средство может быть признано как требующее обслуживание в случаях:

  • Последняя дата обслуживания ОС плюс периодичность обслуживания из норматива обслуживания меньше даты окончания планирования обработки в настройках заполнения
  • У ОС отсутствуют заказы на обслуживание и дата ввода в эксплуатацию плюс периодичность обслуживания из норматива обслуживания меньше даты окончания планирования обработки в настройках заполнения

  • Фактическое значение выработки ОС больше значения выработки по нормативу

 

Автоматически рассчитываются необходимые даты формирования заказов на обслуживание ОС. Заполняется дерево значений, оно имеет 2 уровня:

  • Услуга на обслуживание ОС

  • Дата начала обслуживания ОС, отредактированная пользователем

 

При активизации строк дерева устанавливаются соответствующие отборы на табличное поле справа, содержащее информацию о основных средствах требующих обслуживания.

Пользователь может изменять дату обслуживания, на которую будут формироваться заказы на обслуживание ОС. Расчетная дата обслуживания определяется программой и изменяться пользователем не может, отображается информативно и на формирование заказов на обслуживание ОС влияния не оказывает. Изначально, «расчетная дата обслуживания» и «дата обслуживания» - совпадают.

При автоматическом заполнении табличного поля, программа определяет нормативы обслуживания ОС. Количество норматива определяется из справочника и может корректироваться пользователем для подстановки в формируемые заказы на обслуживание ОС.

В поле «Последнее обслуживание» отображается дата последнего обслуживания ОС по текущему нормативу, если дата последнего обслуживания отображается красным цветом - последний заказ на обслуживание ОС еще не завершен окончательно, синим цветом отображается дата последнего обслуживания в случае если нет ни одного проведенного заказа на обслуживание ОС.

В колонке «Выработка» отображается текущее значений выработки ОС параметра из норматива.

Если ОС требуется обслуживание исходя их временных параметров (периодичности обслуживания ОС по нормативу и даты последнего обслуживания), то фон ячейки строки в поле «Периодичность» будет выделен цветом. Если ОС требуется обслуживание по превышению значения параметра выработки - то фон ячеек строки в полях «Периодичность» и «Выработка» будет выделен цветом.

Пользователь может добавлять даты обслуживания в дерево, перетаскивать ОС из табличного поля в другие даты обслуживания в пределах одной услуги на обслуживание ОС.

Флажками в табличном поле необходимо отметить ОС, по которым пользователь хочет сформировать заказы на обслуживание.

При нажатии на кнопку «Выполнить» производится формирования заказов на обслуживание ОС по заданным пользователем параметрам, открывается список сформированных заказов и перезаполняются данные дерева и табличного поля обработки с учетом сформированных и проведенных заказов.
