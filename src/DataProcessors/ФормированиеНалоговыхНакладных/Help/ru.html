Обработка "Формирование налоговых документов"

   Обработка "Формирование налоговых документов" предназначена для помощи пользователю в заполнении номенклатурным составом налоговых документов (Налоговая накладная и Приложение 2 к налоговой накладной ).  В качестве исходных данных выступают суммы налоговых обязательств по НДС, рассчитанные по данным учета. Расчет производится на текущий или на указанный пользователем момент времени, либо по датам за произвольный период. Номенклатурный состав  формируемых налоговых документов переносится из указанных пользователем документов-оснований (это могут быть счета-фактуры, документы реализации товаров и услуг, и т.д.). Важной особенностью данной обработки является возможность интерактивно отбирать номенклатуру из документов оснований,  изменять количество по каждой строке номенклатуры, как произвольным образом, так и с использованием сервисных алгоритмов (см. ниже).

   На закладке "Настройка" пользователь выбирает режим работы обработки и устанавливает фильтры для расчета неподтвержденных налоговых обязательств. Обязательным для заполнения является реквизит Организация. Дата, вид расчетов (расчеты или возврат), список контрагентов или договоров указывать не обязательно. 

   Существует два режима работы:

"Обязательства, возникшие за период" -  налоговые обязательства будут определяются с точностью до дня, однако в случае одновременного осуществления операций с разными ставками НДС (или разными счетами учета НДС) рассчитанные налоговые обязательства будут представлены без указания дополнительных разрезов, аналогично обязательствам по авансовым оплатам.
Существующие (проведенные) налоговые накладные будут показаны в таблице номенклатурного состава (см. ниже) жирным шрифтом.
Проверку корректности выписки налоговых документов необходимо осуществлять используя отчеты Проверка суммы обязательств по НДС и Отчет о суммах выписанных налоговых документов.

"Неподтвержденные обязательства на дату" - налоговые обязательства определяются на указанный момент времени, при этом учитываются выписанные (проведенные) до указанной даты налоговые документы.

  После установки фильтров на закладке "Настройка" и нажатии кнопки "Заполнить" происходит расчет сумм неподтвержденных налоговых обязательств по данным регистра "НДС Продаж" с учетом момента возникновения налоговых обязательств, которые задаются для каждого договора контрагента, и происходит переход ко второй закладке - "Данные". 

   На закладке "Данные" расположены две таблицы. В верхней таблице "Суммы неподтвержденных налоговых обязательств" отображается информация о суммах неподтвержденных налоговыми документам и обязательствах по НДС.  Обновление сумм (расчет по данным учета) происходит по нажатию кнопки "Обновить".  Имеется возможность добавлять, удалять и вручную корректировать данные таблицы - для этого необходимо перейти на закладку "Дополнительно". Необходимо учитывать, что работа сервисных алгоритмов по изменению товарного состава (см. ниже) основывается не на данных учета, а на суммах, указанных в данной таблице.   В данной таблице кроме информации о контрагенте, договоре контрагента, суммах по каждой из ставок НДС по расчетам за товары и сумме по таре отображается следующая информация:

"Расчеты/возврат" -  признак указывает на то, что обязательства необходимо подтверждать либо по прямым операциям ("расчеты") либо по операциям возврата. В первом случае подтверждающим документом является Налоговая накладная, во втором - Приложение 2 к налоговой накладной.

Метод определения базы - момент определения возникновения налоговых обязательств по НДС (может принимать значения "По оплате", "По первому событию" и т. д.). Устанавливается в договоре контрагента отдельно для операций взаиморасчетов по таре и по товарам.

Аванс - сумма оплат, которые не закрыты отгрузками, и на которые не выписаны подтверждающие налоговые документы. Заметим, что при оплатах в системе не регистрируется ставка НДС и сумма налога, указанная в назначении платежа.

Дата - в зависимости от режима работы отображается либо дата возникновения налоговых обязательств либо дата расчета неподтвержденных налоговых обязательств.

   Все суммы в таблицах отображаются в валюте взаиморасчетов с контрагентом, суммы включают НДС.

    В нижнюю таблицу "Номенклатурный состав формируемых налоговых документов" пользователь вносит документы, часть номенклатурного состава которых будет перенесен в налоговые документы. Кроме того, у пользователя имеется возможность дополнительно ввести номенклатурный состав вручную. Для этого необходимо дважды щелкнуть мышкой по строке "Строки, добавленные вручную". Открывшаяся форма позволит пользователю указать  произвольный товарный состав, который будет дополнительно использован при формировании налоговых документов (заметим, что  суммы включают НДС и указываются в валюте договора контрагента).  Эта возможность отсутствует, если текущий вид расчетов "возврат". Причина в том, что  в качестве подтверждающих налоговые обязательства документов выступают документы "Приложение 2 к налоговой накладной", а указание в этих документах исходных налоговых накладных является обязательным. Поэтому, для заполнения номенклатурным составом Приложения 2 требуется использовать только данные Налоговых накладных.

   При добавлении документа-основания в таблицу "Номенклатурный состав формируемых налоговых документов" весь номенклатурный состав помечается для перенесения в налоговый документ. Документы из данной таблицы станут документами-основаниями в формируемых налоговых документах (если на закладке "Настройки" не установлен флаг "Формировать итоговые накладные"). При двойном щелчке мыши на строке с документом открывается форма изменения номенклатурного состава. Кроме обычных реквизитов в каждой табличной части имеются следующие реквизиты: "Количество в документ", "Сумма в документ" и "Сумма НДС в документ" (последнего реквизита нет в табличной части "Тара").  Эти реквизиты доступны для изменения, а их значения считаются помеченными для перенесения в формируемые налоговые документы. В каждой табличной части имеется возможность устанавливать значения этих реквизитов "в ноль" либо устанавливать равными значениям самого документа-основания. Заметим, что суммы на форме изменения номенклатурного состава отображаются в валюте документа-основания.

  Документы-основания можно добавить вручную, либо воспользоваться подбором.

   На закладке "Дополнительно" отображается уточненная информация - развернутая в разарезе видов налоговой деятельности и счетов учета НДС. Информация относится к текущей строке таблицы неподтвержденных налоговых обязательств на закладке "Данные".

   Сервисные алгоритмы изменения товарного состава

   В обработке имеются несколько сервисных возможностей по групповому изменению товарного состава формируемых налоговых документов. Одновременно изменяется номенклатурный состав по всем документам-основаниям выделенного в верхней таблице договора. Заметим, что строки номенклатурного состава, указанного пользователем вручную не изменяются.  Выполнение обработки происходит при выборе одного из пунктов меню кнопки "Изменить":

Пропорционально под сумму обязательств (вся номенклатура)  - количество номенклатуры, предназначенное для перенесения в налоговый документ по каждому документу-основанию, будет уменьшено пропорционально под текущие суммы неподтвержденных налоговых обязательств. Сумма аванса при этом будет пропорционально учтена по каждой из ставок НДС, встречающихся в документах-основаниях. После данной обработки по каждой строке номенклатуры количество, помеченное для перенесения в налоговый документ, будет не больше количества из документа-основания.

Пропорционально под сумму обязательств (только выбранная номенклатура) - аналогично предыдущему варианту. Отличие состоит в том, что обработка затронет лишь те позиции номенклатуры, у которых на момент обработки отмеченное для перенесения в налоговый документ количество отлично от нуля.

C учетом выписанных налоговых документов - после данной обработки количество, помеченное для перенесения в налоговый документ, станет равным количеству из документа-основания за вычетом количества, на которое уже были выписаны налоговые документы (в которых данный документ был указан в качестве документа-основания).

Заметим, что алгритмы учитывают суммы документов не только в разрезе ставок НДС, но и в разрезе видов налоговой деятельности и счетов учета НДС.

   Формирование налоговых документов по всем договорам, указанным в верхней таблице закладки "Данные" используя номенклатурный состав документов, указанных в нижней таблице этой же закладки, осуществляется по нажатию на кнопку "Сформировать". Если суммы неподтвержденных обязательств с учетом аванса по одной или нескольким ставкам НДС, видам налоговой деятельности или счетов учета НДС  меньше сумм, указанных в таблице номенклатурного состава - будет выдано соответствующее предупреждение.

   Имеется возможность открывать, проводить и распечатывать сформированные налоговые документы. Эти опции устанавливаются соответствующими флажками и переключателями на закладке "Настройка".
