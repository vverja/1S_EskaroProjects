Документ «Закрытие заказов на производство»

Документ «Закрытие заказов на производство» предназначен для принудительного закрытия заказов на производство. Документ позволяет:

  • закрыть сам заказ, включая потребность заказа на производство;
  • снять резервы материалов и полуфабрикатов на складах под заказ на производство;
  • снять размещение материалов и полуфабрикатов (потребности заказа) в заказах поставщикам, внутренних заказах, заказах на производство;
  • снять размещение продукции из заказа покупателя, внутреннего заказа в заказе на производство.

 

Для выполнения закрытия заказов на производство в документе следует выбрать вид операции «закрытие заказов на производство». В этом случае будет  закрыта неисполненная часть заказа, а также сняты все резервы и размещения, в которых участвуют указанные в документе заказы. Если в конфигурации ведется учет потребностей заказов на производство, то все неудовлетворенные потребности также будут закрыты.

Если в документе выбран вид операции «снятие резервов и размещения», то будет выполнено только снятие всех резервов и размещений, в которых участвуют указанные в документе заказы, а также будут закрыты все неудовлетворенные потребности заказов, если ведется учет потребностей.

 

Заполнение документа может происходить тремя способами: ручным подбором в табличную часть документов, подлежащих закрытию; введением на основании документа «Заказ на производство»; с помощью автоматизированного отбора документов по кнопке «Заполнить».

Для операции «снятие резервов и размещения» в автоотборе доступен только один вид заполнения:

  • Все заказы с резервами. Отбираются все документы с наличием резервов и/или размещения по данному заказу. Размещены ли в данном заказе другие заказы - при данном виде заполнения не учитывается.

     

    Для операции «закрытие заказов на производство» в автоотборе доступны следующие предустановленные виды заполнения:

    • Устаревшие заказы. Отбираются заказы, по которым нет резервов и размещений, в которых не размещены другие заказы, которые не выполнены, и истек «срок давности». Истечение срока определяется по «дате исполнения», установленной в заказах. Срок давности по умолчанию установлен [90 дней] и может быть изменен пользователем в пределах от 0 до 999 дней.
    • Исполненные заказы. Исполненными считаются те заказы, по которым: нет резервов, в которых не размещены другие заказы, по которым отсутствуют потребности, по которым выпущена вся продукция. Считается, что все работы по таким заказам успешно завершены и заказы подлежат закрытию. 
    • Произвольный отбор. Позволяет пользователю самостоятельно определить все условия отбора, включая недоступные в других видах заполнения.

     

    Отбор проводится по следующим параметрам:

    • наличие резервов;
    • наличие потребностей;
    • наличие затрат в НЗП;
    • наличие размещения в заказе на производство;
    • состояние исполнения;
    • дата исполнения;

    на закладке «Отбор» формы можно настроить подбор заказов на производство по реквизитам, а также свойствам и категориям документов.

     

    При использовании отбора по «дате исполнения» расчет срока давности (просроченность даты) выполняется с учетом значения параметра «Способ контроля дней задолженности» в настройках параметров учета:

    • либо по календарным дням
    • либо по рабочим дням (согласно регламентированному производственному календарю)


    Для каждого закрываемого заказа на производство может быть указана причина закрытия. На основании данных о причинах закрытия заказов можно сформировать отчет «Анализ причин закрытия заказов».
