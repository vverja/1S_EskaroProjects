Документ «Оплата от покупателя платежной картой»

Документ предназначен для регистрации факта оплаты платежной картой от юридического лица или возврата денежных средств покупателю в случае оплаты платежной картой.

В документе предусмотрены два вида операции:

  • Оплата от покупателя. Для регистрации факта оплаты от контрагента  платежной картой.
  • Возврат денежных средств покупателю. Для регистрации факта возврата денежных средств контрагенту, если оплата была сделана платежной картой.

Закладка  «Основная»

На закладке «Основная» вводится информация для отражения оплаты или возврата денежных средств во взаиморасчетах с контрагентом, который производит оплату или которому возвращаются денежные средства.

Возможны два варианта отражения поступления оплаты (возврата оплаты):

  • по одному договору, сделке, расчетному документу;
  • по нескольким договорам, сделкам, расчетным документам.

Отражение поступления (возврата) оплаты по одному договору, сделке, расчетному документу

При отражении поступления (возврата) оплаты по одному договору,сделке, расчетному документу в полях ввода формы документа необходимо указать:

  • договор с конрагентом - договор с покупателем, комиссионером или комитентом;
  • сделку (если взаиморасчеты ведутся в разрезе сделок , то обязательно, иначе при необходимости);
  • расчетный документ (если в договоре установлен флажок «Вести по документам расчетов с конрагентом»);
  • сумму платежа (изменяется автоматически при редактировании суммы документа);
  • курс валюты взаиморасчетов по договору, по которому будет учтен платеж во взаиморасчетах или сумму, на которую изменится состояние взаиморасчетов;
  • проект (при необходимости);
  • для проведения по бухгалтерскому учету - ставку НДС, статью движения денежных средств, счет учета расчетов с контрагентом и счет учета расчетов с контрагентом по авансам.

Предусмотрена возможность выбора договора, сделки и расчетного документа для погашения из списка текущих задолженностей по указанному контрагенту. Список открывается при нажатии на кнопку «Подбор».

Отражение поступления (возврата) оплаты по нескольким договорам, сделкам, расчетным документам

В этом режиме становится доступной для редактирования табличная часть «Расшифровка платежа». Для каждой строки табличной части необходимо указать:

  • договор с контрагентом - договор с покупателем, комиссионером или комитентом;
  • сделку (если взаиморасчеты ведутся в разрезе сделок , то обязательно, иначе при необходимости);
  • расчетный документ (если в договоре установлен флажок «Вести по документам расчетов с конрагентом»);
  • сумму платежа (часть суммы документа, относимую на данный договор, сделку, расчетный документ); итог по колонке «Сумма платежа» должен равняться сумме документа; 
  • курс валюты взаиморасчетов по договору, по которому будет учтен платеж во взаиморасчетах или сумму, на которую изменится состояние взаиморасчетов;
  • проект (при необходимости);
  • для проведения по бухгалтерскому учету - ставку НДС, статью движения денежных средств, счет учета расчетов с контрагентом и счет учета расчетов по авансам.

Предусмотрена возможность подбора договоров, сделок, расчетных документов для погашения из списка текущих задолженностей по указанному контрагенту. Список открывается при нажатии на кнопку «Подбор».

Табличную часть «Расшифровка платежа» можно автоматически заполнить на основании информации по непогашенным задолженностям. Обработка для автоматического заполнения вызывается при нажатии на кнопку «Заполнить». В окне настройки автоматического заполнения необходимо указать:

  • способ расчета задолженности
    • по фактической задолженности (на основании данных о фактическом движении средств и изменении взаиморасчетов с контрагентами);
    • по оперативной задолженности (помимо данных о фактическом изменении расчетов указываются заказы покупателей и поставщикам, неоплаченные расчетные денежные документы, отраженные в оперативном учете, а также заявки на расходование средств и планируемые поступления денежных средств);
  • порядок погашения задолженности: указывается, какие задолженности будут подбираться для погашения в первую очередь: более ранние или более поздние.
  • общую сумму, на которую будут подбираться задолженности при установке соответствующего флага.

Закладка «Эквайринг»

На закладке «Эквайринг» вводится информация для отражения взаиморасчетов с банком-эквайрером.

В реквизите «Договор эквайринга» выбирается договор эквайринга на обслуживание операций продажи с оплатой платежными картами, заключенный с банком-эквайрером. По выбранному договору эквайринга автоматически определяется банк и договор с контрагентом, по которому будет отражены взаиморасчеты с банком, данная информация отображается в форме документа.

В  реквизите «Вид оплаты» выбирается наименование платежной карты, которой покупатель произвел оплату. В соответствии с выбранным договорм эквайринга и платежной картой определяется процент торговой уступки и рассчитывается сумма торговой уступки. Эта информация отображается в форме документа справочно и не влияет на сумму взаиморасчетов, указанную в документе.

В реквизите «Счет расчетов» выбирается счет расчетов с банком-эквайрером.
