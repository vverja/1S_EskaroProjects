Документ «Требование-накладная»

Документ предназначен для оформления операции передачи материалов, товаров, продукции, полуфабрикатов со склада в производство или списания на затраты.

 

В одном документе можно отразить любые затраты : 

  • Производственные;
  • Брак в производстве;
  • Общепроизводственные;
  • Административные;
  • Затраты на сбыт;
  • Вложения во внеоборотные активы;
  • Прочие операционные затраты ;
  • Прочие;
  • Не учитываемые в управленческом учете.

 Реквизиты шапки документа

  • Склад - склад, с которого произведена выдача ТМЦ в производство, склад также может быт указан в табличной части, однако если в табличной части значение склада не указано то используется значение склада из шапки документа.
  • Подразделение - подразделение в управленческом учете, на затраты которого следует отнести стоимость ТМЦ.
  • Подразделение организации - подразделение организации, на затраты которого следует отнести списание ТМЦ. Аналитика по подразделениям организаций указана для всех счетов учета производственных затрат.
  • Проект. Проект, по которому будут отражены затраты по списанным ТМЦ. Необходимость ведения учета затрат по проектам настраивается в учетной политике на закладке «Расчет себестоимости и учет затрат» в реквизите «Ведение учета затрат по проектам». Учет по проектам возможен для затрат с характером затрат:  «Общепроизводственные расходы», «Административные расходы», «Затраты на сбыт», «Прочие». Для затрат с характером затрат «Производственные расходы», «Брак в производстве» и «Вложения во внеоборотные активы» учет по проектам не ведется. Проект указывается либо в шапке документа, либо в табличной части. Это зависит от значения реквизита «Указание проектов в табличной части документов» в настройке параметров учета.

 

Закладка «Материалы»

 

Указывается перечень материалов, предаваемых со склада в подразделение. 

  • Номенклатура, Характеристика номенклатуры. Передаваемые со склада в производство ТМЦ (материалы, полуфабрикаты, бракованная продукция), указываются с точностью до характеристики.
  • Серия номенклатуры - серия списываемых ТМЦ, указывается, если для номенклатуры задана необходимость вести учет по сериям, и такой складской учет действительно ведется; с помощью кнопки «Заполнить и провести» можно автоматически заполнить информацию о сериях номенклатуры в соответствии с текущими остатками товаров на складах, после заполнения данных документ автоматически проводится.
  •  К.мест, Мест, Ед.мест, Количество, Ед., К., «Количество мест» - вводится, если в первичном документе указано количество мест, а не количество ТМЦ. В этом случае необходимо также указать единицу измерения места. Программа выполняет расчет количества ТМЦ и подставляет в строку в реквизит «Количество». Рядом выводится единица измерения этого количества - единица хранения номенклатуры. Выводится также найденный коэффициент пересчета между единицами измерения. В учете операция отражается по количеству в единице хранения.
  • В т.ч. отпущено сверх лимита. Количество ТМЦ, которое отпускается сверх установленного лимита. Для того чтобы пользователь имел возможность указывать отпуск материалов сверх лимита в наборе дополнительных прав значение права «Редактирование разрешения превышения лимита отпуска материалов» должно быть установлено в «Истина». В этом случае пользователю в шапке документа «Требование-накладная» становится доступным флажок «Разрешить превышение лимита». Если флажок будет включен, то документом можно будет списать любое количество материалов на затраты независимо от значения установленного лимита. В этом случае общее количество списываемого материала должно быть указано в реквизите «Количество», а объем превышающий значение лимита должен быть явно выделен и указан в колонке «В т.ч. отпущено сверх лимита». Колонка становится видимой при установке флажка «Разрешить превышение лимита».
  • Заказ затраты. Заказ, на который относят затраты. Распределение косвенных затрат если для них задан заказ, выполняется только на выпуск продукции по заданному заказу. В качестве заказа могут быть указаны документы «Заказ покупателя» и «Заказ на производство».
  • Заказ резерв. Указывается в случае, если номенклатуру надо списать из резерва по данному заказу. В качестве заказа резерва могут быть указаны документы «Заказ покупателя», «Заказ на производство» или «Внутренний заказ».
  • Внутренний заказ. Указывается в случае, если номенклатура списывается для выполнения данного внутреннего заказа.
  • Статья затрат - указывается статья затрат по отражению расходов. С точки зрения выбора статьи затрат, важно отметить, что при списании следует выбирать статьи затрат с признаком принадлежности именно материальной группе. Далее в зависимости от того, на что пойдут материалы, можно оформить их списание либо на производственные расходы, подразумевая, что в дальнейшем эти материалы будут включаться в себестоимость продукции как прямые расходы, либо списать на другие виды косвенных расходов. Если материалы списываются на производственные расходы, то они включаются в объем незавершенного производства, иначе затраты к незавершенному производству относится не будут. Для информации характер затрат по выбранной статье затрат показан в реквизите «Хар-р затрат».
  • Счет учета - необходимо указать счет учета ТМЦ, с которого производится списание стоимости передаваемых ТМЦ. Заполняется, только если документ отражается в БУ.
  • Счет затрат, Аналитика. Счет затрат, на который по бухгалтерскому учету будут отнесены затраты и аналитика согласно выбранному счету. Информативно вид аналитики для выбранного счета показан в реквизите «Вид аналитики».
  • Проект. Проект, по которому будут отражены затраты по списанным ТМЦ. Необходимость ведения учета затрат по проектам настраивается в учетной политике на закладке «Расчет себестоимости и учет затрат» в реквизите «Ведение учета затрат по проектам». Учет по проектам возможен для затрат с характером затрат: «Общепроизводственные расходы», «Административные расходы», «Затраты на сбыт», «Прочие». Для затрат с характером затрат «Производственные расходы», «Брак в производстве» и «Вложения во внеоборотные активы» учет по проектам не ведется. Проект указывается либо в шапке документа, либо в табличной части. Это зависит от значения реквизита «Указание проектов в табличной части документов» в настройке параметров учета.
  • Качество. Качество передаваемых ТМЦ.
  • Склад. Склад, с которого производится передача ТМЦ в производство. Если в табличной части значение склада не указано, то используется значение склада из шапки документа.

Табличную часть документа можно заполнить автоматически с помощью кнопки «Заполнить». Предусмотрены следующие варианты заполнения:

  • Заполнить по внутреннему заказу. В документ добавляются данные о материалах из выбранного пользователем внутреннего заказа, причем оценивается только невыполненная часть заказа. Реквизит «Внутренний заказ» заполняется ссылкой на выбранный внутренний заказ. Если на складах есть материалы зарезервированные под выбранный внутренний заказ, то заполнение производится с учетом резервов (заполняются реквизиты  «Заказ резерв» и «Склад»). При этом:
      • если количество резерва превышает остаток по внутреннему заказу – в документ попадает количество в соответствии с остатком по внутреннему заказу;
      • если количество резерва меньше остатка по внутреннему заказу – в документ добавляются две строки, одна с заполненными реквизитами «Заказ резерв» и «Склад», а другая с пустыми значениями этих реквизитов, на общее количество невыполненной части внутреннего заказа.

       Аналогичным образом заполняется документ «Требование-накладная» и при вводе его на основании внутреннего заказа.

  • Заполнить по заказу покупателя (переработка). В документ добавляются данные о материалах из выбранного пользователем заказа покупателя с видом «переработка», причем оценивается только невыполненная часть заказа.
  • Добавить из поступления. Добавляются строки из выбранного пользователем документа «Поступление товаров и услуг».
  • Заполнить по заданию на производство. Пользователем выбирается документ «Задание на производство». По перечню продукции, указанному в документе, выполняется разузлование по спецификации до уровня материалов. Итоговый список материалов добавляется в документ «Требование-накладная».
  • Заполнить по потребностям. Пользователем выбирается документ «Заказ на производство». По перечню материалов, указанному в документе «Заказ на производство» на закладке «Материалы», отбираются те материалы, которые зарезервированы на складах под этот заказ. Итоговый список материалов добавляется в документ «Требование-накладная».

 

Особенности проведения документа

 

При проведении документа по товарам, для которых требуется сертификация, производится проверка ее наличия. В случае, если нет внешней или внутренней сертификации, выводятся сообщения и документ не проводится. Данное ограничение касается только списания на производственные затраты.

 

Непосредственно при проведении документа будет выполняться анализ превышения объемов списываемых материалов над установленным лимитом. Контроль превышения будет выполняться только для данных управленческого учета. Если лимиты будут превышены, то система выдаст сообщение об этом и документ не будет проведен. Контроль превышения лимитов для данных бухгалтерского учета не ведется.
