Документ «Формирование потребностей»

Документ предназначен для формирования общего объема потребностей в товарах и материалах.

Информацию, которая содержится в документе, можно разделить на три категории:

  • источники спроса;
  • текущие потребности;
  • сформированные потребности.

Источниками спроса могут быть:

  • планы продаж;
  • планы производства;
  • заказы покупателей;
  • внутренние заказы;
  • заказы на производство.

Текущими потребностями могут быть:

  • планы закупок;
  • посменные планы производства.

Данные по каждому виду источников спроса и текущих потребностей вводятся на отдельной закладке.

Планы продаж

Табличная содержит перечень планов продаж, по составу которых будет извлекаться потребность. Данный перечень не содержит ссылок на документы «План продаж», а определяет состав показателей, ограничивающих выборку оборотов из регистра накопления «Планы продаж». Обязательными для заполнения реквизитами являются «Сценарий», «Дата нач.» и «Дата кон.». Для дополнительного уточнения выборки можно указать проект и подразделение. Значения, указанные в реквизитах «Проект» и «Подразделение» участвуют в ограничении выборки. Выборка данных по планам продаж будет осуществляться в интервале определенном значениями реквизитов «Дата нач.» и «Дата кон.».

Заказы покупателей

Табличная содержит перечень планов заказов покупателей, по неисполненным частям которых будет извлекаться потребность. Выборка данных производится из остатков регистра накопления «Заказы покупателей» на дату документа «Формирование потребностей». Обязательным для заполнения реквизитом является «Дата потребности». При выборе значения реквизита «Заказ» дата потребности устанавливается равной дате отгрузки, указанной в заказе. Если значение реквизита «Заказ» не заполнено, то выбираются все неисполненные заказы.

Формирование потребностей

Последовательность механизма формирования потребностей:

  • По перечню показателей для источников спроса (указанных в табличных частях «Планы продаж», «Планы производства», «Заказы покупателей», «Внутренние заказы», «Заказы на производство») осуществляется выборка данных из соответствующих регистров накопления. В выборку не попадают позиции номенклатуры с типом номенклатуры «Услуга».
  • Номенклатурные группы, полученные из укрупненных планов производства или продаж, распределяются по номенклатуре в соответствии с весами вхождения номенклатуры в номенклатурную группу.
  • Выполняется корректировка полученных потребностей:
    • потребности уменьшаются на величину резервов по заказам покупателей, внутренним заказам, заказам на производство (остатки по регистру накопления «Товары в резерве на складах» на дату документа).
    • потребности уменьшаются на величину остатков товаров на складах (остатки по регистру «Товары на складах» на дату документа). Из остатков товаров на складах исключаются:
      • резервы по заказам покупателей, внутренним заказам, заказам на производство (остатки по регистру накопления «Товары в резерве на складах» на дату документа);
      • остатки тары (разность остатков по регистрам накопления «Товары полученные» и «Товары переданные» со статусом передачи «Возвратная тара» на дату документа);
      • товары к передаче со складов (остатки по регистру накопления «Товары к передаче со складов» на дату документа).
  • Исключаются полученные отрицательные потребности.
  • Для позиции с видом воспроизводства «Производство», выполняется разузлование до материалов. При этом существует такое правило: позиции, полученные из заказов на производство, всегда имеют вид воспроизводства «Производство», для позиций, полученных из других источников, используется вид воспроизводства, указанный в справочнике «Номенклатура». Если спецификация номенклатуры явно указана в источнике потребности, то при разузловании используется эта спецификация, иначе используется основная спецификация.
  • По перечню показателей для текущих потребностей (указанных в табличных частях «Планы закупок», «Посменные планы производства») осуществляется выборка данных из соответствующих регистров накопления.
  • Складываются результаты выборки по источникам спроса (с учетом выполненных корректировок) и выборки по текущим потребностям.
  • В зависимости от режима выбора показателей, указанного в документе на закладке «Дополнительно», выполняется сложение или объединение данных. При сложении позиции с равными значениями показателей «Дата потребности», «Номенклатура», «Характеристика номенклатуры», «Заказ», «Конечная продукция», «Проект» и «Товар тара» сворачиваются в одну, значение показателя «Количество» суммируется. При объединении позиции по такому же принципу сворачиваются в одну, но значение показателя «Количество» принимает значение максимального из объединяемых.
  • Полученными потребностями заполняется табличная часть на закладке «Потребности».

Сформированные автоматически потребности можно скорректировать вручную.

При проведении документа состав табличной части «Потребности» будет записан в регистр сведений «Календарные потребности в номенклатуре», также в этот регистр будут записаны движения, обнуляющие ранее сформированные другими документами потребности. То есть потребности, сформированные более ранними документами, считаются удовлетворенными.   

Данные из регистра сведений «Календарные потребности в номенклатуре» используются для формирования заказов поставщикам с помощью обработки «Календарный план закупок».

Замечание. Данные о потребностях хранятся  в регистре «Календарные потребности в номенклатуре» в единицах хранения остатков. При автоматическом формировании потребностей данные учитываются в единицах хранения остатков, не зависимо от того, какие единицы измерения были указаны в источниках спроса. Если данные о потребности вносятся в табличную часть «Потребности» вручную, то для позиции номенклатуры можно указать единицу измерения отличную от единицы хранения остатков. При проведении документа для таких позиций будет выполнен пересчет в единицы хранения остатков.
