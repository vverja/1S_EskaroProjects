Отчет «Анализ заказа покупателя»

Отчет «Анализ заказа покупателя» предназначен для получения информации о состоянии заказа покупателя на момент последнего движения по заказу или на указанную дату. Отчет вызывается из формы документа «Заказ покупателя» при нажатии на кнопку «Анализ». В отчете показывается информация о состоянии отгрузки по заказу, об источниках обеспечения заказа товарами, а также о состоянии взаиморасчетов с покупателем.

В части отгрузки по заказу выводится количество товара в единицах измерения, указанных в заказе.

  • Запланировано – количество товара, которое указано в заказе покупателя.
  • Отгружено/отменено - количество отгруженного товара с учетом имеющихся корректировок по заказу.
  • Осталось отгрузить – количество товара, которое осталось отгрузить по заказу.

В части комплектации заказа выводятся показатели: 

  • Резерв - то количество товара, которое зарезервировано на складах под заказ.
  • Заказано – то количество товара, которое заказано у поставщиков.
  • Свободный остаток - то количество товара, которое числится на складах и может быть использовано для обеспечения заказа.
  • Заказать – то количество товара, которое следует дополнительно зарезервировать или разместить в заказах поставщиков для того, чтобы исполнить заказ. При расчете этого количества не учитывается свободный остаток товара на складе - предполагается, что пользователь самостоятельно принимает решение об использовании свободного складского остатка.

Суммы оплаты и отгрузки по заказу выводятся с учетом тех корректировок, которые были введены по заказу. В части оплаты товаров по заказу выводятся следующие показатели.

  • Запланировано – сумма запланированной оплаты по заказу, в том числе:
    1. Всего - та сумма, которая указана в заказе покупателя с учетом введенных корректировок по заказу.
    2. Предоплата % – та сумма, которую покупатель должен оплатить в соответствии с указанным в договоре процентом предоплаты. Например, сумма по заказу покупателя равна 1000 рублей, процент предоплаты 50%. Тогда в этой графе будет отображена сумма 500 рублей. 
  • Оплачено - сумма поступившей по заказу оплаты.
  • Оплатить – та сумма, которую необходимо погасить, в том числе: 
    1. Всего - та сумма, которую осталось погасить, чтобы полностью оплатить заказ.
    2. Предоплата – та сумма предоплаты, которую покупателю осталось внести в соответствии с указанным в договоре процентом предоплаты. Например, сумма по заказу покупателя равна 1000 рублей, процент предоплаты 50%. Покупатель оплатил 400 рублей. Тогда в этой графе будет отображена сумма 100 рублей. 

Справочно выводится информация о состоянии расчетов по другим заказам в рамках договора, указанного в документе «Заказ покупателя». Состояние расчетов с покупателем отражается в отчете только в том случае, если по договору установлен контроль суммы задолженности.

Для документа «Заказ покупателя» с видом операции «Переработка» в отчет выводятся данные по материалам, полученным в переработку.

  • Запланировано – количество материалов, которое указано в заказе покупателя.
  • Получено/отменено - количество полученных материалов с учетом имеющихся корректировок по заказу.
  • Осталось получить – количество материалов, которое осталось получить от заказчика.

 

Отчет «Анализ заказа поставщику»

Отчет предназначен для анализа состояния заказу поставщику на момент последнего движения или на указанную дату. Отчет вызывается из формы документа «Заказ поставщику» при нажатии на кнопку «Анализ». В отчете показывается информация о состоянии поступления по заказу, его размещении, а также о состоянии взаиморасчетов с поставщиком.

Суммы оплаты и поступления по заказу выводятся с учетом тех корректировок, которые были введены по заказу.

В части поступления по заказу выводится количество товара в единицах измерения, указанных в заказе.

  • Запланировано – количество товара, которое указано в заказе поставщику.
  • Получено/отменено - количество полученного товара с учетом имеющихся корректировок по заказу.
  • Осталось получить – количество товара, которое осталось получить по заказу.

В части размещения заказа выводятся показатели: 

  • Поступает в резерв по заказам - то количество товара, которое зарезервировано под заказы покупателей.
  • Поступает на склад – то количество товара, которое поступает в качестве свободного остатка.

В части оплаты товаров по заказу выводятся следующие показатели.

  • Запланировано – сумма, которая планируется к оплате поставщику по заказу. 
  • Оплачено - сумма, оплаченная поставщику по данному заказу (в случае если используется ведение взаиморасчетов по заказам).
  • Оплатить – та сумма, которую необходимо погасить.

Справочно выводится информация о состоянии расчетов по другим заказам в рамках договора, указанного в документе «Заказ поставщику», а также информация о необходимости сертификации по товарно-материальным ценностям, поступившим под заказ.

Для документа «Заказ поставщику» с видом операции «Переработка» в отчет выводятся данные по материалам, передаваемым в переработку.

  • Запланировано – количество материалов, которое указано в заказе поставщику.
  • Отгружено/отменено - количество переданных материалов с учетом имеющихся корректировок по заказу.
  • Осталось отгрузить – количество материалов, которое осталось передать поставщику в переработку.

 

Отчет «Анализ внутреннего заказа»

Отчет предназначен для анализа состояния внутреннего заказа на момент последнего движения или на указанную дату. Отчет вызывается из формы документа «Внутренний заказ» при нажатии на кнопку «Анализ». В отчете выводятся показатели в части выполнения заказа, а также о состоянии источников его обеспечения.

Количество, запланированное к выполнению по заказу, выводится с учетом тех корректировок, которые были введены по заказу.

В части выполнения заказа выводится количество товара в единицах измерения, указанных в заказе. 

  • Запланировано – количество товаров, которое указано во внутреннем заказе с учетом корректировок, оформленных по заказу.
  • Выполнено/отменено - количество товара, по которому выполнен заказ, с учетом имеющихся корректировок по заказу.
  • Осталось выполнить – общее количество товаров, которое осталось выполнить по заказу.

В части комплектации заказа выводятся показатели: 

  • Резерв - то количество товара, которое зарезервировано на складах под заказ.
  • Заказано – то количество товара, которое заказано у поставщиков, в заказах на производство и других внутренних заказах.
  • Свободный остаток - то количество товара, которое числится на складах и может быть использовано для обеспечения заказа.
  • Заказать – то количество товара, которое следует дополнительно зарезервировать или разместить в заказах для того, чтобы исполнить заказ. При расчете этого количества не учитывается свободный остаток товара на складе - предполагается, что пользователь самостоятельно принимает решение об использовании свободного складского остатка.
